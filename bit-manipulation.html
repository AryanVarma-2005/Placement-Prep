Bits & Binary
How computers actually store numbers — the foundation of everything
ConceptWhat is a Bit?

A bit (binary digit) is the smallest unit of data in a computer. It can only be 0 or 1. Every number, character, image, and program is stored as a sequence of bits.

8 bits = 1 byte. A 32-bit integer uses 32 bits. A 64-bit integer uses 64 bits.

Decimal 13 = Binary 1101 = Hex 0xD

1×2³ + 1×2² + 0×2¹ + 1×2⁰
 = 8   + 4   + 0   + 1 = 13

Number Representation
Two's complement, signed integers, overflow — what Python does differently
ConceptTwo's Complement (How CPUs store negatives)

Negative numbers are stored using two's complement: flip all bits, then add 1. This is why ~n = -(n+1) in Python.

How to get -5 from +5 (8-bit example):
+5:0000 0101
Flip bits:1111 1010 (= ~5 in 8-bit)
Add 1:1111 1011 (= -5 in 8-bit two's complement)✅
Verify:0000 0101 + 1111 1011 = 1 0000 0000 = 0 (overflow discarded)

# Let n = 44 = 0b00101100

# ── GET bit at position i ──────────────────────
def get_bit(n, i):
    return (n >> i) & 1
# get_bit(44, 3) = (44>>3)&1 = 5&1 = 1  ✅ (bit 3 is set)
# get_bit(44, 0) = (44>>0)&1 = 44&1 = 0 (bit 0 not set)

# ── SET bit at position i (force to 1) ─────────
def set_bit(n, i):
    return n | (1 << i)
# set_bit(44, 0) = 44|1 = 0b00101100|0b00000001 = 0b00101101 = 45

# ── CLEAR bit at position i (force to 0) ───────
def clear_bit(n, i):
    return n & ~(1 << i)
# clear_bit(44, 3) = 44 & ~8 = 101100 & 110111 = 100100 = 36

# ── TOGGLE bit at position i ────────────────────
def toggle_bit(n, i):
    return n ^ (1 << i)
# toggle_bit(44, 3) = 44^8 = 101100^001000 = 100100 = 36
# toggle_bit(44, 0) = 44^1 = 101100^000001 = 101101 = 45

# ── UPDATE bit to a given value v (0 or 1) ─────
def update_bit(n, i, v):
    return (n & ~(1 << i)) | (v << i)
# update_bit(44, 2, 0) → clear bit 2
# update_bit(44, 0, 1) → set bit 0

# ── READ a range of bits [hi..lo] ───────────────
def get_bits(n, lo, hi):
    mask = (1 << (hi - lo + 1)) - 1
    return (n >> lo) & mask
# get_bits(44, 2, 4) → bits 2-4 of 44
# 44 = 0b101100 → bits[4:2] = 011 = 3

Counting Bits & Parity
Count set bits efficiently — from O(n) to O(1) to O(1) with lookup
CodeCounting Set Bits — 4 Methods
Python
# METHOD 1: Built-in (Python 3.10+) — O(1) hardware
n.bit_count()       # fastest
bin(n).count('1')  # Pythonic, works everywhere

# METHOD 2: Brian Kernighan's Algorithm — O(set bits)
def count_bits_bk(n):
    count = 0
    while n:
        n &= (n - 1)  # Remove lowest set bit each iteration
        count += 1
    return count
# For n=44 (101100): 3 iterations → count=3

# METHOD 3: Naive — O(log n)
def count_bits_naive(n):
    count = 0
    while n:
        count += n & 1  # Check last bit
        n >>= 1         # Shift right
    return count

# METHOD 4: DP — count bits for 0..n in O(n)
def countBits(n):
    """LeetCode 338: count bits for each number 0..n"""
    dp = [0] * (n + 1)
    for i in range(1, n + 1):
        # dp[i] = dp[i >> 1] + (i & 1)
        # i's bits = (i/2)'s bits + 1 if i is odd
        dp[i] = dp[i >> 1] + (i & 1)
    return dp
# countBits(5) = [0,1,1,2,1,2]

# PARITY: is number of 1-bits even or odd?
def parity(n):
    result = 0
    while n:
        result ^= 1  # XOR with 1 toggles 0↔1
        n &= (n - 1)
    return result
# parity(44) = 1 (44 has 3 set bits = odd → parity 1)
LeetCode 338Counting Bits — DP with Bit Trick
countBits(5): explain dp[i] = dp[i>>1] + (i&1)
i=0:0 in binary → dp[0]=0
i=1:dp[1>>1]=dp[0]=0, 1&1=1 → dp[1]=1
i=2:dp[2>>1]=dp[1]=1, 2&1=0 → dp[2]=1
i=3:dp[3>>1]=dp[1]=1, 3&1=1 → dp[3]=2
i=4:dp[4>>1]=dp[2]=1, 4&1=0 → dp[4]=1

Key insight: i >> 1 removes last bit. (i & 1) adds back whether last bit was 1. So dp[i] = bits in i/2 + last bit of i.

CodePowers of Two — Complete Toolkit
Python
# ── Is n a power of 2? ──────────────────────────
def is_power_of_two(n):
    return n > 0 and (n & (n - 1)) == 0
# Why? Powers of 2 have exactly one 1-bit:
# 8  = 1000
# 7  = 0111
# 8 & 7 = 0  ← magic!
# 6  = 0110
# 5  = 0101
# 6 & 5 = 4 ≠ 0  ← not power of 2

# ── Next power of 2 ≥ n ────────────────────────
def next_power_of_two(n):
    if n <= 0: return 1
    n -= 1
    n |= n >> 1
    n |= n >> 2
    n |= n >> 4
    n |= n >> 8
    n |= n >> 16
    return n + 1
# next_power_of_two(5) = 8, next_power_of_two(8) = 8

# ── Log base 2 (position of highest set bit) ───
def log2(n):
    return n.bit_length() - 1
# log2(8) = 3, log2(5) = 2, log2(1) = 0

# ── Fast modulo (only for power of 2 m) ─────────
n % m   # general
n & (m - 1)   # O(1) fast version IF m is power of 2!
# n % 8  ==  n & 7  (8 = 2^3, mask = 0b111)
# n % 4  ==  n & 3
# n % 16 ==  n & 15

# ── Power of 4 check ────────────────────────────
def is_power_of_four(n):
    # Must be power of 2 AND the 1-bit is at an even position
    # 0x55555555 = 01010101... (1s at even positions)
    return n > 0 and (n & (n-1)) == 0 and (n & 0x55555555) != 0

XOR Patterns
XOR's magical properties unlock some of the most elegant DSA solutions
PropertiesThe 5 XOR Properties That Matter
P1:a ^ 0 = a    (XOR with 0 → identity)a=a
P2:a ^ a = 0    (XOR with self → zero)cancels!
P3:a ^ b = b ^ a  (commutative)
P4:(a^b)^c = a^(b^c)  (associative)
P5:a ^ b ^ a = b  (self-cancellation → find unique)KEY!
💡
P2+P5 are why XOR finds the unique element
XOR all numbers together. Pairs cancel out (a^a=0). Only the unique number survives!
[1,2,3,2,1] → 1^2^3^2^1 = (1^1)^(2^2)^3 = 0^0^3 = 3
CodeXOR — The Single Number Family
Python
# ═══════════════════════════════════════
# CLASSIC: Single Number I — LC 136
# Every element appears twice except one
# ═══════════════════════════════════════
def singleNumber(nums):
    result = 0
    for n in nums:
        result ^= n  # Pairs cancel: a^a=0
    return result   # O(n) time, O(1) space!

# ═══════════════════════════════════════
# HARDER: Single Number III — LC 260
# Two unique numbers, rest appear twice
# ═══════════════════════════════════════
def singleNumber3(nums):
    xor = 0
    for n in nums:
        xor ^= n  # xor = a ^ b (the two unique numbers)

    # Find any bit where a and b differ (lowest set bit of xor)
    diff_bit = xor & (-xor)

    # Partition numbers into two groups by this bit
    a = 0
    for n in nums:
        if n & diff_bit:
            a ^= n  # Group 1: bit is set
    return [a, xor ^ a]  # [a, b]

# ═══════════════════════════════════════
# XOR SWAP — no temp variable
# ═══════════════════════════════════════
a, b = 5, 9
a ^= b   # a = 5^9
b ^= a   # b = 9^(5^9) = 5
a ^= b   # a = (5^9)^5 = 9
# Now a=9, b=5  ← swapped!

# ═══════════════════════════════════════
# Missing Number — LC 268
# ═══════════════════════════════════════
def missingNumber(nums):
    result = len(nums)
    for i, n in enumerate(nums):
        result ^= i ^ n  # XOR with index AND value
    return result
# [3,0,1]: result = 3^(0^3)^(1^0)^(2^1) = 2

Bitmask Dynamic Programming
Represent states as integers — solve exponential problems in O(2ⁿ × n)
ConceptWhat is Bitmask DP?

When n is small (≤ 20), we can represent a subset of n items as an integer. Bit i = 1 means item i is in the set. This turns exponential state enumeration into compact O(2ⁿ × n) DP.

n=4 items: {A, B, C, D}

mask = 0b1010 = 10 → means {B, D} selected
mask = 0b1111 = 15 → means {A, B, C, D} all selected
mask = 0b0000 = 0 → means {} empty set

Total states: 2⁴ = 16 possible subsets
CodeTravelling Salesman Problem (TSP) — Classic Bitmask DP
Python
def tsp(dist):
    """
    TSP: visit all n cities, return to start, min cost.
    Bitmask DP: dp[mask][i] = min cost to visit cities
    in 'mask', ending at city i.
    Time: O(2^n * n^2) | Space: O(2^n * n)
    """
    n = len(dist)
    INF = float('inf')
    # dp[mask][i] = min cost to reach city i
    #               having visited exactly the cities in mask
    dp = [[INF] * n for _ in range(1 << n)]
    dp[1][0] = 0  # Start at city 0, mask = 0001

    for mask in range(1 << n):
        for u in range(n):
            if not (mask >> u) & 1: continue  # u not in mask
            if dp[mask][u] == INF: continue
            for v in range(n):
                if (mask >> v) & 1: continue   # v already visited
                new_mask = mask | (1 << v)
                dp[new_mask][v] = min(
                    dp[new_mask][v],
                    dp[mask][u] + dist[u][v]
                )

    full = (1 << n) - 1  # All cities visited
    return min(dp[full][i] + dist[i][0] for i in range(n))

# Bitmask DP KEY OPERATIONS:
# Check if city i is in mask:     (mask >> i) & 1
# Add city i to mask:              mask | (1 << i)
# Remove city i from mask:         mask & ~(1 << i)
# Check all cities visited:        mask == (1 << n) - 1
CodePartition to K Equal Subsets — LC 698
Python
def canPartitionKSubsets(nums, k):
    """
    Can nums be partitioned into k equal-sum subsets?
    Bitmask DP: dp[mask] = True if subset represented
    by mask can be distributed into complete buckets.
    """
    total = sum(nums)
    if total % k != 0: return False
    target = total // k
    n = len(nums)
    dp = [False] * (1 << n)
    dp[0] = True
    cur_sum = [0] * (1 << n)

    for mask in range(1 << n):
        if not dp[mask]: continue
        for i in range(n):
            if (mask >> i) & 1: continue  # i already used
            next_mask = mask | (1 << i)
            if cur_sum[mask] + nums[i] <= target:
                dp[next_mask] = True
                cur_sum[next_mask] = (cur_sum[mask] + nums[i]) % target
    return dp[(1 << n) - 1]

Subsets & Enumeration

CodeEnumerate All Subsets with Bitmask

# ── All 2^n subsets ───────────────────────────────
def all_subsets(arr):
    n = len(arr)
    result = []
    for mask in range(1 << n):   # 0 to 2^n - 1
        subset = []
        for i in range(n):
            if (mask >> i) & 1:  # Bit i is set in mask
                subset.append(arr[i])
        result.append(subset)
    return result
# all_subsets([1,2,3]):
# mask=000 → []
# mask=001 → [1]    (bit 0 set)
# mask=010 → [2]    (bit 1 set)
# mask=011 → [1,2]  (bits 0,1 set)
# ... 8 total subsets

# ── Enumerate all submasks of a mask ──────────────
# Useful for subset-sum, partition problems
def all_submasks(mask):
    sub = mask
    while sub > 0:
        # Process sub (a submask of mask)
        yield sub
        sub = (sub - 1) & mask  # Next submask trick!
# For mask=0b1010: yields 0b1010, 0b1000, 0b0010
# Time: O(3^n) to enumerate all submasks for all masks

# ── Count set bits in mask (population count) ─────
popcount = lambda x: bin(x).count('1')

# ── Check if mask2 is a subset of mask1 ───────────
def is_subset(mask1, mask2):
    return (mask1 & mask2) == mask2

def reverseBits(n):
    """Reverse 32-bit unsigned integer's bits. LC 190."""
    result = 0
    for _ in range(32):
        result = (result << 1) | (n & 1)  # Add last bit of n
        n >>= 1                            # Shift n right
    return result
# reverseBits(43261596) = 964176192
# 00000010100101000001111010011100
# → 00111001011110000010100101000000

# Pythonic one-liner:
def reverseBits2(n):
    return int(bin(n)[2:].zfill(32)[::-1], 2)

def grayCode(n):

# ── XOR of all numbers from 1 to n ────────────────
def xor_to_n(n):
    # Pattern repeats every 4: n%4: 0→n, 1→1, 2→n+1, 3→0
    patterns = [n, 1, n + 1, 0]
    return patterns[n % 4]
# xor_to_n(5) = 5^4^3^2^1 = 1 (use formula, not loop!)

# ── Sum of two integers WITHOUT + operator ────────
def getSum(a, b):
    while b != 0:
        carry = (a & b) << 1  # Carry bits
        a = a ^ b              # Sum without carry
        b = carry
    return a
# getSum(3, 5) = 8
# a=3(011), b=5(101)
# carry=(011&101)<<1 = (001)<<1 = 010
# a=011^101=110=6, b=010
# carry=(110&010)<<1=(010)<<1=100
# a=110^010=100=4, b=100
# carry=(100&100)<<1=1000, a=100^100=0, b=1000
# carry=(0&1000)<<1=0, a=0^1000=8, b=0  → 8 ✅

# ── Divide without / operator ──────────────────────
def divide(dividend, divisor):
    sign = -1 if (dividend < 0) ^ (divisor < 0) else 1
    a, b = abs(dividend), abs(divisor)
    result = 0
    while a >= b:
        temp, mult = b, 1
        while a >= temp << 1:
            temp <<= 1; mult <<= 1
        a -= temp; result += mult
    return sign * result

9. Bitwise AND of Numbers Range
MediumLC 201
Explain
Code

Key Insight: AND of [m..n] = common prefix of m and n in binary. Shift both right until equal — count shifts. Shift result back left by same count.

Python
def rangeBitwiseAnd(m, n):
    shift = 0
    while m != n:
        m >>= 1; n >>= 1; shift += 1
    return m << shift  # O(log n)

11. Find the Duplicate Number
MediumLC 287
Explain
Code

Bit Pattern: For each bit position, count bits set in nums vs 1..n. If nums has more → that bit is set in duplicate.

Python
def findDuplicate(nums):
    n = len(nums) - 1; duplicate = 0
    for bit in range(n.bit_length() + 1):
        mask = 1 << bit
        cnt_nums = sum(1 for x in nums if x & mask)
        cnt_ref  = sum(1 for x in range(1,n+1) if x & mask)
        if cnt_nums > cnt_ref:
            duplicate |= mask
    return duplicate  # O(n log n)
12. Subsets
MediumLC 78
Code
Python
def subsets(nums):
    n = len(nums); result = []
    for mask in range(1 << n):
        subset = [nums[i] for i in range(n) if (mask>>i)&1]
        result.append(subset)
    return result  # O(2^n * n)
13. Maximum XOR of Two Numbers
MediumLC 421
Explain
Code

Pattern: Greedy from MSB to LSB. Try to set each bit of result to 1. Use prefix set: if a^b = candidate with bit set, then a and b both in prefix set.

Python
def findMaximumXOR(nums):
    max_xor = mask = 0
    for i in range(31, -1, -1):
        mask |= 1 << i
        prefixes = {n & mask for n in nums}
        candidate = max_xor | (1 << i)
        if any(candidate ^ p in prefixes for p in prefixes):
            max_xor = candidate
    return max_xor  # O(32n)
14. Total Hamming Distance
MediumLC 477
Explain
Code

Pattern: For each bit position, count ones and zeros. Contribution = ones × zeros. Sum over all 32 bit positions. O(32n) = O(n).

Python
def totalHammingDistance(nums):
    total = 0
    for bit in range(32):
        ones = sum((n >> bit) & 1 for n in nums)
        zeros = len(nums) - ones
        total += ones * zeros  # pairs with different bits
    return total  # O(32n) = O(n)
15. Gray Code
MediumLC 89
Code
Python
def grayCode(n):
    return [i ^ (i >> 1) for i in range(1 << n)]
# Formula: gray(i) = i XOR (i >> 1)
# Each adjacent pair differs by exactly 1 bit.
