01
What is Dynamic Programming?
Understand the core idea before touching any problem
ConceptThe Big Idea

Dynamic Programming is an optimization technique that solves a problem by breaking it into overlapping subproblems, solving each subproblem once, and storing the results to avoid redundant computation.

Two essential properties must hold for DP to apply:

1
Optimal Substructure
The optimal solution to the problem can be built from optimal solutions to its subproblems. (e.g., shortest path, max profit)
2
Overlapping Subproblems
The same subproblems are solved multiple times in a naive recursive approach. DP stores them to avoid recomputation.
AnalogyThe Real-World Intuition

Imagine you're climbing stairs and want to count ways to reach step n (taking 1 or 2 steps at a time). A naive approach recalculates ways(n-1) and ways(n-2) over and over. DP says: write the answer on a sticky note so you never compute it twice.

🧠
Memorize this distinction
Divide and Conquer = subproblems are independent (merge sort).
DP = subproblems overlap (Fibonacci). The same overlap that makes recursion exponential makes DP linear!
FrameworkThe 5-Step DP Framework
1
Define the State
What does dp[i] or dp[i][j] represent? This is the hardest step. Be precise.
2
Write the Recurrence
How does dp[i] relate to dp[i-1], dp[i-2], etc.? Express the transition.
3
Identify Base Cases
What are the smallest subproblems with known answers?
4
Determine Order
Bottom-up (tabulation) or top-down (memoization)? Which subproblems depend on which?
5
Optimize Space (Optional)
Can you reduce the dp array from O(n²) to O(n) or O(1) by only keeping recent rows/values?

⚡ Quick Check
Which property is required for DP but NOT divide-and-conquer?
✅ Overlapping Subproblems! Divide and conquer (merge sort, quick sort) breaks problems into independent subproblems. DP's subproblems overlap — they share computation, which is why memoization helps.

Memoization (Top-Down DP)
Add a cache to recursion — turn exponential into polynomial
ConceptHow Memoization Works

Memoization = recursion + a cache (dict/array). Before computing, check if result is cached. After computing, store the result. That's it.

CodeFibonacci: Naive vs Memoization vs Tabulation
Python
# ══ NAIVE RECURSION — O(2^n) time ══
def fib_naive(n):
    if n <= 1: return n
    return fib_naive(n-1) + fib_naive(n-2)
# fib(40) makes ~2^40 calls! ≈ 1 trillion

# ══ MEMOIZATION — O(n) time, O(n) space ══
def fib_memo(n, memo={}):
    if n in memo: return memo[n]   # Cache hit!
    if n <= 1: return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]

# Cleaner with @lru_cache (Python's built-in memoize)
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)

# ══ TABULATION — O(n) time, O(n) space ══
def fib_tab(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[0], dp[1] = 0, 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# ══ SPACE OPTIMIZED — O(n) time, O(1) space ══
def fib_opt(n):
    if n <= 1: return n
    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b
💡
When to Use Memoization vs Tabulation
Use memoization when: the recursion is natural, not all subproblems need solving, or the dependency order is unclear.
Use tabulation when: all subproblems must be solved, recursion depth is a concern (Python default limit ≈ 1000), or you want to optimize space.
03
Tabulation (Bottom-Up DP)
Fill a table iteratively from base cases upward
ConceptHow Tabulation Works

Tabulation fills a DP table starting from the smallest subproblems (base cases) and builds up to the answer. No recursion — just loops. The key questions are: what do the rows/columns represent? and what order do I fill the table?

💰 COIN CHANGE — TABULATION VISUALIZER

CodeCoin Change — Classic DP
Python
def coinChange(coins, amount):
    """
    State: dp[i] = min coins to make amount i
    Recurrence: dp[i] = min(dp[i-c]+1) for all coins c
    Base: dp[0] = 0 (0 coins needed for amount 0)
    Time: O(amount * len(coins)) | Space: O(amount)
    """
    INF = float('inf')
    dp = [INF] * (amount + 1)
    dp[0] = 0  # Base case

    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)

    return dp[amount] if dp[amount] != INF else -1

# coinChange([1,3,5], 11) = 3  (5+5+1 = 11, 3 coins)
# coinChange([2], 3)      = -1 (impossible)

# DRY RUN for coins=[1,3,5], amount=6:
# dp[0]=0 (base)
# dp[1]=dp[0]+1=1   (use coin 1)
# dp[2]=dp[1]+1=2   (use coin 1)
# dp[3]=min(dp[2]+1, dp[0]+1)=1  (use coin 3!)
# dp[4]=min(dp[3]+1, dp[1]+1)=2  (use coin 3)
# dp[5]=min(dp[4]+1, dp[2]+1, dp[0]+1)=1 (use coin 5!)
# dp[6]=min(dp[5]+1, dp[3]+1, dp[1]+1)=2 (5+1 or 3+3)
⚡ Quick Check
For Coin Change with coins=[1,5,6] and amount=10, what is dp[10]?
✅ dp[10]=2! Use two 5-coins. dp[5]=1 (use coin 5), dp[10]=dp[5]+1=2. Greedy (6+1+1+1+1) would give 5, but DP finds the optimal 5+5=2 coins.

Space Optimization
Reduce O(n²) to O(n) or O(1) — critical for interviews
CodeSpace Optimization Patterns
Python — Space Optimization
# ══ PATTERN 1: Keep only last k values ══
# House Robber — only need dp[i-1] and dp[i-2]
def rob(nums):
    prev2 = prev1 = 0
    for n in nums:
        curr = max(prev1, prev2 + n)
        prev2, prev1 = prev1, curr
    return prev1   # O(1) space instead of O(n)

# ══ PATTERN 2: Keep only last 2 rows (2D → 1D) ══
# Unique Paths — reduce O(m*n) table to O(n) array
def uniquePaths(m, n):
    dp = [1] * n  # One row only
    for _ in range(1, m):
        for j in range(1, n):
            dp[j] += dp[j-1]  # dp[j] = from above, dp[j-1] = from left
    return dp[-1]

# ══ PATTERN 3: In-place update ══
# Triangle — update from bottom to top in-place
def minimumTotal(triangle):
    dp = triangle[-1][:]  # Start with bottom row
    for row in range(len(triangle)-2, -1, -1):
        for j in range(len(triangle[row])):
            dp[j] = triangle[row][j] + min(dp[j], dp[j+1])
    return dp[0]

# ══ KEY RULE: Direction of iteration matters! ══
# If dp[i] depends on dp[i-c] (coin change, unbounded):
#   → iterate LEFT TO RIGHT (use new value)
# If dp[i][j] depends on dp[i-1][j-w] (0/1 knapsack):
#   → iterate RIGHT TO LEFT (use old value, avoid reuse)

CodeClimbing Stairs, Jump Game, House Robber
Python — 1D DP Classics
# ══ CLIMBING STAIRS (LC 70) ══
# State: dp[i] = ways to reach step i
# Recurrence: dp[i] = dp[i-1] + dp[i-2]
def climbStairs(n):
    if n <= 2: return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b   # Exactly Fibonacci! O(n), O(1)

# ══ HOUSE ROBBER (LC 198) ══
# State: dp[i] = max money robbing houses 0..i
# Recurrence: dp[i] = max(dp[i-1], dp[i-2]+nums[i])
# (skip current house OR rob it + best from i-2)
def rob(nums):
    prev2 = prev1 = 0
    for n in nums:
        prev2, prev1 = prev1, max(prev1, prev2 + n)
    return prev1   # O(n), O(1)

# ══ JUMP GAME (LC 55) ══
# Greedy is simpler but DP also works
def canJump(nums):
    max_reach = 0
    for i, n in enumerate(nums):
        if i > max_reach: return False
        max_reach = max(max_reach, i + n)
    return True

# ══ JUMP GAME II (LC 45) — min jumps ══
def jump(nums):
    jumps = curr_end = curr_far = 0
    for i in range(len(nums) - 1):
        curr_far = max(curr_far, i + nums[i])
        if i == curr_end:
            jumps += 1
            curr_end = curr_far
    return jumps

# ══ DECODE WAYS (LC 91) ══
# How many ways to decode a string of digits? (1→A, 26→Z)
def numDecodings(s):
    if not s or s[0] == '0': return 0
    n = len(s)
    dp = [0] * (n + 1)
    dp[0] = 1  # Empty string
    dp[1] = 1  # Single digit (not '0')
    for i in range(2, n + 1):
        one  = int(s[i-1])       # Single digit
        two  = int(s[i-2:i])     # Two digits
        if   1 <= one <= 9:  dp[i] += dp[i-1]
        if  10 <= two <= 26: dp[i] += dp[i-2]
    return dp[n]

Knapsack — 0/1 & Unbounded
The most fundamental 2D DP pattern
Concept0/1 Knapsack — The Core Pattern

Problem: n items with weights and values. Knapsack capacity W. Maximize total value without exceeding capacity. Each item can be used at most once.

State definition is everything:
dp[i][w]= max value using first i items with capacity w
Skip:dp[i][w] = dp[i-1][w] (don't take item i)
Take:dp[i][w] = dp[i-1][w-wt[i]] + val[i] (take item i)
Answer:dp[i][w] = max(Skip, Take) if wt[i] <= w

# ══ 0/1 KNAPSACK — each item used at most once ══
def knapsack01(weights, values, W):
    n = len(weights)
    dp = [[0] * (W + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for w in range(W + 1):
            # Don't take item i
            dp[i][w] = dp[i-1][w]
            # Take item i (if it fits)
            if weights[i-1] <= w:
                dp[i][w] = max(dp[i][w],
                               dp[i-1][w - weights[i-1]] + values[i-1])

    return dp[n][W]  # O(n*W) time, O(n*W) space

# Space-optimized 0/1 knapsack (1D dp, iterate RIGHT TO LEFT)
def knapsack01_opt(weights, values, W):
    dp = [0] * (W + 1)
    for i in range(len(weights)):
        for w in range(W, weights[i] - 1, -1):  # ← RIGHT TO LEFT!
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[W]   # O(n*W) time, O(W) space

# ══ UNBOUNDED KNAPSACK — each item used unlimited times ══
# Coin change is unbounded knapsack!
def unboundedKnapsack(weights, values, W):
    dp = [0] * (W + 1)
    for w in range(1, W + 1):
        for i in range(len(weights)):
            if weights[i] <= w:
                dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[W]   # LEFT TO RIGHT (reuse allowed)

# KEY DIFFERENCE:
# 0/1: loop items OUTSIDE, iterate weight RIGHT TO LEFT
# Unbounded: iterate weight LEFT TO RIGHT (reuse new value)

# ══ SUBSET SUM — can we make sum W from array? ══
def canMakeSum(nums, target):
    dp = [False] * (target + 1)
    dp[0] = True
    for n in nums:
        for w in range(target, n - 1, -1):  # RIGHT TO LEFT
            dp[w] = dp[w] or dp[w - n]
    return dp[target]
🎯 Knapsack Interview Insight
  • 0/1 Knapsack (RIGHT TO LEFT) — each item once: LC 416 (Partition Equal Subset Sum), LC 494 (Target Sum)
  • Unbounded Knapsack (LEFT TO RIGHT) — reuse items: LC 322 (Coin Change), LC 518 (Coin Change 2)
  • The direction of the inner loop is the ONLY difference in the 1D version
  • Subset Sum = 0/1 Knapsack with boolean dp

# ══ LCS — Longest Common Subsequence (LC 1143) ══
# State: dp[i][j] = LCS of text1[:i] and text2[:j]
def longestCommonSubsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:  # Match!
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    return dp[m][n]  # O(m*n) time+space

# ══ LIS — Longest Increasing Subsequence (LC 300) ══
# State: dp[i] = LIS ending at index i
def lengthOfLIS(nums):
    if not nums: return 0
    n = len(nums)
    dp = [1] * n  # Every element is LIS of length 1
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)  # O(n²). O(n log n) with patience sort

# O(n log n) LIS using binary search
import bisect
def lengthOfLIS_fast(nums):
    tails = []
    for n in nums:
        pos = bisect.bisect_left(tails, n)
        if pos == len(tails): tails.append(n)
        else: tails[pos] = n
    return len(tails)

# ══ EDIT DISTANCE (LC 72) ══
# State: dp[i][j] = min ops to convert word1[:i] to word2[:j]
def minDistance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1): dp[i][0] = i  # Delete all
    for j in range(n + 1): dp[0][j] = j  # Insert all

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i-1] == word2[j-1]:
                dp[i][j] = dp[i-1][j-1]  # No operation needed
            else:
                dp[i][j] = 1 + min(
                    dp[i-1][j],    # Delete from word1
                    dp[i][j-1],    # Insert into word1
                    dp[i-1][j-1]  # Replace
                )
    return dp[m][n]

Python — Grid DP
# ══ UNIQUE PATHS (LC 62) ══
# Robot moves only right or down in m×n grid
def uniquePaths(m, n):
    dp = [[1] * n for _ in range(m)]  # Edges all 1
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i-1][j] + dp[i][j-1]  # From top + from left
    return dp[m-1][n-1]

# ══ UNIQUE PATHS II — with obstacles (LC 63) ══
def uniquePathsWithObstacles(grid):
    m, n = len(grid), len(grid[0])
    dp = [[0]*n for _ in range(m)]
    dp[0][0] = 1 if grid[0][0] == 0 else 0
    for i in range(m):
        for j in range(n):
            if grid[i][j] == 1: dp[i][j] = 0; continue
            if i > 0: dp[i][j] += dp[i-1][j]
            if j > 0: dp[i][j] += dp[i][j-1]
    return dp[m-1][n-1]

# ══ MINIMUM PATH SUM (LC 64) ══
def minPathSum(grid):
    m, n = len(grid), len(grid[0])
    dp = [[0]*n for _ in range(m)]
    dp[0][0] = grid[0][0]
    for j in range(1, n): dp[0][j] = dp[0][j-1] + grid[0][j]
    for i in range(1, m): dp[i][0] = dp[i-1][0] + grid[i][0]
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])
    return dp[m-1][n-1]

# ══ MAXIMAL SQUARE (LC 221) ══
# State: dp[i][j] = side of largest square with bottom-right at (i,j)
def maximalSquare(matrix):
    m, n = len(matrix), len(matrix[0])
    dp = [[0]*n for _ in range(m)]
    max_side = 0
    for i in range(m):
        for j in range(n):
            if matrix[i][j] == '1':
                if i == 0 or j == 0: dp[i][j] = 1
                else: dp[i][j] = min(dp[i-1][j],dp[i][j-1],dp[i-1][j-1])+1
                max_side = max(max_side, dp[i][j])
    return max_side * max_side

dp[i][j] = answer for subarray/string from i to j

Python — Interval DP
# ══ PALINDROMIC SUBSTRINGS (LC 647) ══
# State: dp[i][j] = True if s[i..j] is palindrome
def countSubstrings(s):
    n = len(s)
    dp = [[False]*n for _ in range(n)]
    count = 0
    for length in range(1, n+1):       # length of interval
        for i in range(n - length + 1):   # start index
            j = i + length - 1              # end index
            if s[i] == s[j] and (length <= 2 or dp[i+1][j-1]):
                dp[i][j] = True; count += 1
    return count

# ══ BURST BALLOONS (LC 312) — Hard interval DP ══
# Add 1 at each end: [1] + nums + [1]
# State: dp[i][j] = max coins from balloons i+1..j-1
# Key trick: k is the LAST balloon to burst in (i,j)
def maxCoins(nums):
    nums = [1] + nums + [1]
    n = len(nums)
    dp = [[0]*n for _ in range(n)]

    for length in range(2, n):          # interval length
        for i in range(n - length):       # left boundary
            j = i + length
            for k in range(i+1, j):       # k = last burst
                coins = nums[i]*nums[k]*nums[j] + dp[i][k] + dp[k][j]
                dp[i][j] = max(dp[i][j], coins)

    return dp[0][n-1]

# ══ LONGEST PALINDROMIC SUBSEQUENCE (LC 516) ══
def longestPalindromeSubseq(s):
    n = len(s)
    dp = [[0]*n for _ in range(n)]
    for i in range(n): dp[i][i] = 1  # Each char is palindrome of len 1
    for length in range(2, n+1):
        for i in range(n - length + 1):
            j = i + length - 1
            if s[i] == s[j]: dp[i][j] = dp[i+1][j-1] + 2
            else: dp[i][j] = max(dp[i+1][j], dp[i][j-1])
    return dp[0][n-1]

Python — Tree DP
# ══ HOUSE ROBBER III (LC 337) — Can't rob parent+child ══
# State: return (rob_root, skip_root) for each node
# rob_root  = best value including this node
# skip_root = best value excluding this node
def rob3(root):
    def dfs(node):
        if not node: return (0, 0)
        left_rob,  left_skip  = dfs(node.left)
        right_rob, right_skip = dfs(node.right)
        # Rob current: can't rob children
        rob  = node.val + left_skip + right_skip
        # Skip current: take best from each child
        skip = max(left_rob, left_skip) + max(right_rob, right_skip)
        return (rob, skip)
    rob, skip = dfs(root)
    return max(rob, skip)

# ══ DIAMETER OF BINARY TREE (LC 543) — Classic tree DP ══
def diameterOfBinaryTree(root):
    max_diam = [0]
    def height(node):
        if not node: return 0
        L, R = height(node.left), height(node.right)
        max_diam[0] = max(max_diam[0], L + R)
        return 1 + max(L, R)
    height(root)
    return max_diam[0]

# ══ BINARY TREE CAMERAS (LC 968) — Greedy+DP on trees ══
# State: 0=not covered, 1=covered (no camera), 2=has camera
# Return minimum cameras to cover all nodes
def minCameraCover(root):
    cameras = [0]
    def dfs(node):
        if not node: return 1  # Null = covered
        left, right = dfs(node.left), dfs(node.right)
        if left == 0 or right == 0:  # Child needs coverage
            cameras[0] += 1; return 2
        if left == 2 or right == 2:  # Child has camera, covers us
            return 1
        return 0  # Not covered — parent must cover us
    if dfs(root) == 0: cameras[0] += 1  # Root uncovered
    return cameras[0]

def maxProfit1(prices):
    min_price = float('inf'); max_profit = 0
    for p in prices:
        min_price = min(min_price, p)
        max_profit = max(max_profit, p - min_price)
    return max_profit

# ══ II: UNLIMITED TRANSACTIONS (LC 122) ══
def maxProfit2(prices):
    # Buy every ascending pair
    return sum(max(prices[i]-prices[i-1],0) for i in range(1,len(prices)))

# ══ III: AT MOST 2 TRANSACTIONS (LC 123) ══
def maxProfit3(prices):
    b1=b2=-float('inf'); s1=s2=0
    for p in prices:
        b1=max(b1,-p)          # Best buy for 1st
        s1=max(s1,b1+p)        # Best sell for 1st
        b2=max(b2,s1-p)        # Best buy for 2nd (use s1 profit)
        s2=max(s2,b2+p)        # Best sell for 2nd
    return s2

# ══ IV: AT MOST K TRANSACTIONS (LC 188) ══
def maxProfit4(k, prices):
    n = len(prices)
    if k >= n // 2: return maxProfit2(prices)  # Unlimited
    buy  = [-float('inf')] * (k + 1)
    sell = [0] * (k + 1)
    for p in prices:
        for j in range(k, 0, -1):
            sell[j] = max(sell[j], buy[j] + p)
            buy[j]  = max(buy[j],  sell[j-1] - p)
    return sell[k]

# ══ COOLDOWN (LC 309) ══
# States: held, sold (cooldown), rest
def maxProfitCooldown(prices):
    held = -float('inf'); sold = rest = 0
    for p in prices:
        held, sold, rest = \
            max(held, rest - p), held + p, max(rest, sold)
    return max(sold, rest)

# ══ WITH FEE (LC 714) ══
def maxProfitFee(prices, fee):
    hold = -prices[0]; cash = 0
    for p in prices[1:]:
        hold = max(hold, cash - p)
        cash = max(cash, hold + p - fee)
    return cash

def climbStairs(n):
    a, b = 1, 2
    for _ in range(3, n+1): a, b = b, a+b
    return b if n >= 2 else n  # O(n), O(1)

5. Partition Equal Subset Sum

Pattern: 0/1 Knapsack. Can we reach target = sum/2? Use boolean dp array, iterate RIGHT TO LEFT.

def canPartition(nums):
    total = sum(nums)
    if total % 2: return False
    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True
    for n in nums:
        for j in range(target, n-1, -1):  # RIGHT to LEFT!
            dp[j] = dp[j] or dp[j-n]
    return dp[target]

8. Word Break
MediumLC 139
Explain
Code

Pattern: 1D DP. dp[i] = can we form s[:i] using wordDict? Check all endpoints j where dp[j] is True and s[j:i] is a word.

Python
def wordBreak(s, wordDict):
    words = set(wordDict)
    n = len(s)
    dp = [False] * (n + 1)
    dp[0] = True
    for i in range(1, n+1):
        for j in range(i):
            if dp[j] and s[j:i] in words:
                dp[i] = True; break
    return dp[n]  # O(n^2) time, O(n) space
9. Longest Palindromic Substring
MediumLC 5
Explain
Code

Pattern: Expand around center OR interval DP. Expand-around-center is O(n) space vs O(n²) for table.

Python
def longestPalindrome(s):
    res = ""
    for i in range(len(s)):
        for odd, even in [(i,i),(i,i+1)]:  # Odd & even centers
            l, r = odd, even
            while l>=0 and r<len(s) and s[l]==s[r]: l-=1;r+=1
            if r-l-1 > len(res): res = s[l+1:r]
    return res  # O(n^2) time, O(1) space
10. Maximum Subarray (Kadane's)
MediumLC 53
Explain
Code

Pattern: Kadane's algorithm — 1D DP. dp[i] = max subarray ending at i = max(nums[i], dp[i-1]+nums[i]). Track global max.

def maxSubArray(nums):
    curr = best = nums[0]
    for n in nums[1:]:
        curr = max(n, curr + n)
        best = max(best, curr)
    return best  # O(n), O(1)

15. Regular Expression Matching
HardLC 10
Explain
Code

Pattern: 2D string DP. Handle '*' by looking 2 positions back. dp[i][j] = does s[:i] match p[:j]?

Python
def isMatch(s, p):
    m, n = len(s), len(p)
    dp = [[False]*(n+1) for _ in range(m+1)]
    dp[0][0] = True
    for j in range(2, n+1):
        if p[j-1] == '*': dp[0][j] = dp[0][j-2]
    for i in range(1, m+1):
        for j in range(1, n+1):
            if p[j-1] in (s[i-1], '.'): dp[i][j] = dp[i-1][j-1]
            elif p[j-1] == '*':
                dp[i][j] = dp[i][j-2]  # Zero occurrences
                if p[j-2] in (s[i-1], '.'): dp[i][j] |= dp[i-1][j]
    return dp[m][n]
16. Russian Doll Envelopes
HardLC 354
Explain
Code

Pattern: Sort + LIS. Sort by width asc, then height DESC (to prevent same-width nesting). Then find LIS on heights only.

Python
import bisect
def maxEnvelopes(envelopes):
    envelopes.sort(key=lambda x: (x[0], -x[1]))
    tails = []
    for _, h in envelopes:
        pos = bisect.bisect_left(tails, h)
        if pos == len(tails): tails.append(h)
        else: tails[pos] = h
    return len(tails)  # O(n log n)

📊 Complexity by Pattern
  • 1D DP (Fibonacci, stairs): O(n), O(1)
  • Knapsack (1D opt): O(n×W), O(W)
  • LCS/Edit Distance: O(m×n), O(m×n)
  • Interval DP: O(n³), O(n²)
  • Bitmask DP: O(2ⁿ×n²), O(2ⁿ×n)
  • LIS (fast): O(n log n), O(n)
🎯 Pattern Recognition
  • "Min/max over choices" → DP
  • "Count ways" → DP (add subproblems)
  • "True/False achievable" → boolean DP
  • "Items with weights + capacity" → Knapsack
  • "Two strings comparison" → LCS/Edit
  • "Interval i..j" → Interval DP
  • "Subset of n≤20" → Bitmask DP
  • "Grid traversal" → Grid DP
⚠️ Common Mistakes
  • Wrong direction (L→R vs R→L for knapsack)
  • Off-by-one in base cases
  • Forgetting to handle impossible case (return -1)
  • State includes wrong information
  • Confusing "at most k" vs "exactly k" transactions
  • Not seeing Fibonacci in "climbing stairs" style
