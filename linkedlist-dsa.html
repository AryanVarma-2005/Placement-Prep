01
Introduction to Linked Lists
What they are, why they exist, and where you see them
ConceptWhat is a Linked List?

A Linked List is a linear data structure where elements (called nodes) are stored in memory and each node points to the next one using a reference (pointer).

Unlike arrays, nodes are NOT stored in contiguous memory. Each node holds two things:

  • data — the actual value
  • next — a reference to the next node (or None for the last node)
AnalogyReal-World Examples
🚂
Train Cars
Each car connects to the next. Remove one car without rebuilding the whole train.
🔗
Chain Links
Each link connects to the next. Insert/remove anywhere by changing connections.
🌐
Browser History
Each page points to the previous. Click back = traverse the list.
🎵
Music Playlist
Each song points to the next. Add/remove songs from anywhere easily.
📝
Undo History
Text editors store operations in a linked list for undo/redo.
💾
File System
OS file allocation tables use linked lists to track disk blocks.

🧠
Memory Trick: LL vs Array
Array = apartment building with numbered rooms (fast access, costly to add/remove rooms). Linked List = houses on a street connected by roads (slow to find house #47, but easy to build/demolish a house between two others).
⚡ Quick Check
What is the time complexity of accessing the kth element in a Linked List?
✅ O(n) worst case! Even though you traverse exactly k steps, k can be n in the worst case. No random access — you must start from head and follow pointers.

Singly Linked List
Build one from scratch — node by node
ConceptThe Node — Building Block

Every linked list is made of nodes. Each node has exactly two parts: a data field and a next pointer.

Python
class Node:
    """A single node in a linked list."""
    def __init__(self, data):
        self.data = data   # The value stored
        self.next = None   # Pointer to next node (None = end)

# Creating individual nodes
n1 = Node(10)
n2 = Node(20)
n3 = Node(30)

# Linking them manually
n1.next = n2  # 10 → 20
n2.next = n3  # 20 → 30
# n3.next is None automatically → end of list

# Traversal: follow .next pointers
current = n1
while current:
    print(current.data, end=" → ")
    current = current.next
# Output: 10 → 20 → 30 →
ImplementationFull Singly Linked List Class
Python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class SinglyLinkedList:
    def __init__(self):
        self.head = None  # Points to first node
        self.tail = None  # Points to last node (O(1) append)
        self.size = 0

    # ── Append to end: O(1) with tail pointer ──
    def append(self, data):
        node = Node(data)
        if not self.head:
            self.head = self.tail = node
        else:
            self.tail.next = node
            self.tail = node
        self.size += 1

    # ── Prepend to front: O(1) ──
    def prepend(self, data):
        node = Node(data)
        node.next = self.head
        self.head = node
        if not self.tail:
            self.tail = node
        self.size += 1

    # ── Insert at index: O(n) ──
    def insert_at(self, index, data):
        if index == 0:
            return self.prepend(data)
        if index >= self.size:
            return self.append(data)
        node = Node(data)
        cur = self.head
        for _ in range(index - 1):  # Go to (index-1)th node
            cur = cur.next
        node.next = cur.next  # New node points to old next
        cur.next = node        # Previous node points to new node
        self.size += 1

    # ── Delete by value: O(n) ──
    def delete(self, data):
        if not self.head:
            return False
        # If head itself is the target
        if self.head.data == data:
            self.head = self.head.next
            if not self.head:
                self.tail = None
            self.size -= 1
            return True
        # Search for node before the target
        cur = self.head
        while cur.next and cur.next.data != data:
            cur = cur.next
        if cur.next:
            if cur.next == self.tail:
                self.tail = cur
            cur.next = cur.next.next  # Bypass the target node
            self.size -= 1
            return True
        return False

    # ── Search: O(n) ──
    def search(self, data):
        cur, idx = self.head, 0
        while cur:
            if cur.data == data:
                return idx
            cur, idx = cur.next, idx + 1
        return -1

    # ── Traverse: O(n) ──
    def __repr__(self):
        nodes, cur = [], self.head
        while cur:
            nodes.append(str(cur.data))
            cur = cur.next
        return " → ".join(nodes) + " → None"

# === Demo ===
ll = SinglyLinkedList()
ll.append(10); ll.append(20); ll.append(30)
ll.prepend(5)
print(ll)          # 5 → 10 → 20 → 30 → None
ll.insert_at(2, 15)
print(ll)          # 5 → 10 → 15 → 20 → 30 → None
ll.delete(15)
print(ll)          # 5 → 10 → 20 → 30 → None
print(ll.search(20))  # 2 (index)
print(ll.size)        # 4

Before:head → [10] → [20] → [30] → None
Insert 25 after 20:Find node with value 20
Step 1:new_node.next = current.next (25 → 30)⚠️ Do this FIRST!

Python — Deletion Patterns
# ── Delete HEAD node ──
def delete_head(head):
    if not head:
        return None
    return head.next  # New head is second node

# ── Delete TAIL node ──
def delete_tail(head):
    if not head or not head.next:
        return None
    cur = head
    while cur.next.next:  # Stop at second-to-last
        cur = cur.next
    cur.next = None  # Disconnect last node
    return head

# ── Delete node with GIVEN VALUE ──
def delete_value(head, val):
    dummy = Node(0)   # Dummy simplifies edge cases
    dummy.next = head
    prev = dummy
    cur = head
    while cur:
        if cur.data == val:
            prev.next = cur.next  # Bypass target node
            return dummy.next
        prev, cur = cur, cur.next
    return dummy.next

# ── Delete Nth node from END ──
def delete_nth_from_end(head, n):
    dummy = Node(0)
    dummy.next = head
    fast = slow = dummy
    # Move fast n+1 steps ahead
    for _ in range(n + 1):
        fast = fast.next
    # Move both until fast reaches end
    while fast:
        fast, slow = fast.next, slow.next
    slow.next = slow.next.next  # Delete nth from end
    return dummy.next
⚠️
The Dummy Node Trick — Learn This!
A dummy node (also called sentinel node) is placed before the head. It eliminates edge cases for head deletion. Instead of handling "what if head is the target," every node now has a predecessor. Use this in 90% of linked list problems!
⚠️
CRITICAL Order of Operations
When inserting: link the new node forward BEFORE updating the previous node. If you do it backwards, you lose the rest of the list! This is the #1 beginner mistake.
⚡ Quick Check
When inserting a node in the middle of a linked list, what must you do FIRST?
✅ Always set new_node.next = current.next FIRST. Then set current.next = new_node. Doing it the other way loses the reference to the rest of the list!
04
Doubly Linked List
Two pointers per node — traverse in both directions
ConceptWhat Makes it "Doubly"?

Each node in a Doubly Linked List has three parts:

  • prev — pointer to the previous node
  • data — the value
  • next — pointer to the next node

This allows bidirectional traversal — you can go forward AND backward. The tradeoff is extra memory (one extra pointer per node).

None ← [prev|10|next] ⇄ [prev|20|next] ⇄ [prev|30|next] → None
ImplementationDoubly Linked List in Python
Python
class DNode:
    """Doubly linked list node."""
    def __init__(self, data):
        self.data = data
        self.prev = None  # ← Backward pointer
        self.next = None  # → Forward pointer

class DoublyLinkedList:
    def __init__(self):
        self.head = None
        self.tail = None
        self.size = 0

    def append(self, data):
        node = DNode(data)
        if not self.tail:
            self.head = self.tail = node
        else:
            node.prev = self.tail  # New node looks back
            self.tail.next = node  # Old tail looks forward
            self.tail = node
        self.size += 1

    def prepend(self, data):
        node = DNode(data)
        if not self.head:
            self.head = self.tail = node
        else:
            node.next = self.head  # New node looks forward
            self.head.prev = node  # Old head looks back
            self.head = node
        self.size += 1

    def delete(self, node):
        """Delete a specific node in O(1) — no traversal needed!"""
        if node.prev:
            node.prev.next = node.next
        else:
            self.head = node.next  # Deleting head
        if node.next:
            node.next.prev = node.prev
        else:
            self.tail = node.prev  # Deleting tail
        self.size -= 1

    def traverse_forward(self):
        cur, res = self.head, []
        while cur:
            res.append(cur.data); cur = cur.next
        return res

    def traverse_backward(self):
        cur, res = self.tail, []
        while cur:
            res.append(cur.data); cur = cur.prev
        return res

# Demo
dll = DoublyLinkedList()
dll.append(10); dll.append(20); dll.append(30)
dll.prepend(5)
print(dll.traverse_forward())   # [5, 10, 20, 30]
print(dll.traverse_backward())  # [30, 20, 10, 5]

# O(1) deletion with direct node reference!
node_to_delete = dll.head.next  # Node with 10
dll.delete(node_to_delete)
print(dll.traverse_forward())   # [5, 20, 30]

Circular Linked List
The last node points back to the first — no dead end
ConceptWhat is Circular?

In a Circular Linked List, the last node's next pointer points back to the head, forming a loop. There is no None at the end.

head → [10] → [20] → [30] → [40] ─┐
          └──────────────────────────────┘

Applications: Round-robin scheduling, multiplayer game turns, music repeat playlists, circular buffers.

ImplementationCircular Linked List
Python
class CNode:
    def __init__(self, data):
        self.data = data
        self.next = None

class CircularLL:
    def __init__(self):
        self.head = None

    def append(self, data):
        node = CNode(data)
        if not self.head:
            self.head = node
            node.next = self.head  # Points to itself!
            return
        # Find last node
        cur = self.head
        while cur.next != self.head:
            cur = cur.next
        cur.next = node
        node.next = self.head  # New last points to head

    def traverse(self, n=None):
        """Traverse n nodes (or all unique nodes if n=None)."""
        if not self.head:
            return []
        result = []
        cur = self.head
        count = 0
        while True:
            result.append(cur.data)
            cur = cur.next
            count += 1
            # Stop if we've gone full circle OR reached n
            if cur == self.head or (n and count >= n):
                break
        return result

    def detect_loop(self):
        """A circular list always has a loop — this is by design."""
        return True  # Circular LL always loops

# Demo
cll = CircularLL()
for v in [10, 20, 30, 40]:
    cll.append(v)

print(cll.traverse())   # [10, 20, 30, 40]
print(cll.traverse(6))  # [10, 20, 30, 40, 10, 20] — wraps!

Two-Pointer Techniques
Fast & slow pointers — the most powerful linked list trick
ConceptWhat are Two Pointers?

Two pointers are two variables that traverse the same linked list, but at different speeds or different starting positions. This lets you solve problems in O(n) time and O(1) space that would otherwise need extra memory.

The most famous pair: Slow (tortoise) moves 1 step, Fast (hare) moves 2 steps.

Python — Two Pointer Templates
# ═══════════════════════════════════════
# PATTERN 1: Find Middle of Linked List
# ═══════════════════════════════════════
def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next        # 1 step
        fast = fast.next.next   # 2 steps
    # When fast reaches end, slow is at middle
    return slow

# Dry Run on [1,2,3,4,5]:
# slow=1,fast=1 → slow=2,fast=3 → slow=3,fast=5
# fast.next=None → exit. slow=3 = MIDDLE ✅

# ═══════════════════════════════════════
# PATTERN 2: Nth from End
# ═══════════════════════════════════════
def nth_from_end(head, n):
    fast = head
    # Move fast n steps ahead
    for _ in range(n):
        fast = fast.next
    slow = head
    # Both move together until fast reaches end
    while fast:
        slow, fast = slow.next, fast.next
    # Now slow is at nth from end
    return slow

# ═══════════════════════════════════════
# PATTERN 3: Intersection of Two Lists
# ═══════════════════════════════════════
def get_intersection(headA, headB):
    a, b = headA, headB
    while a != b:
        # Switch to other list's head when reaching None
        a = a.next if a else headB
        b = b.next if b else headA
    # They meet at intersection (or both None if no intersection)
    return a

# ═══════════════════════════════════════
# PATTERN 4: Check Palindrome
# ═══════════════════════════════════════
def is_palindrome(head):
    # Step 1: Find middle with slow/fast
    slow = fast = head
    while fast and fast.next:
        slow, fast = slow.next, fast.next.next

    # Step 2: Reverse second half
    prev, cur = None, slow
    while cur:
        cur.next, prev, cur = prev, cur, cur.next
    right = prev  # Head of reversed second half

    # Step 3: Compare both halves
    left = head
    while right:
        if left.data != right.data:
            return False
        left, right = left.next, right.next
    return True

CodeAll Reversal Variants
Python — Reversal Patterns
# ═══════════════════════════════════════
# PATTERN 1: Reverse Entire List — O(n) time, O(1) space
# ═══════════════════════════════════════
def reverse(head):
    prev = None
    cur = head
    while cur:
        next_node = cur.next  # Save next (CRITICAL!)
        cur.next = prev       # Reverse the pointer
        prev = cur            # Move prev forward
        cur = next_node       # Move cur forward
    return prev  # prev is new head

# DRY RUN on [1→2→3→4→5]:
# prev=None, cur=1: next=2, 1→None, prev=1, cur=2
# prev=1, cur=2:    next=3, 2→1,    prev=2, cur=3
# prev=2, cur=3:    next=4, 3→2,    prev=3, cur=4
# prev=3, cur=4:    next=5, 4→3,    prev=4, cur=5
# prev=4, cur=5:    next=None,5→4,  prev=5, cur=None
# Return prev=5 → [5→4→3→2→1→None] ✅

# ═══════════════════════════════════════
# PATTERN 2: Reverse Between Positions (LeetCode 92)
# ═══════════════════════════════════════
def reverseBetween(head, left, right):
    dummy = Node(0)
    dummy.next = head
    prev = dummy

    # Move prev to node just before 'left'
    for _ in range(left - 1):
        prev = prev.next

    cur = prev.next  # Start of reversal zone

    # Reverse (right - left) times
    for _ in range(right - left):
        next_node = cur.next
        cur.next = next_node.next     # Bypass next_node
        next_node.next = prev.next    # next_node goes to front
        prev.next = next_node         # prev points to next_node

    return dummy.next

# ═══════════════════════════════════════
# PATTERN 3: Reverse in K-Groups (LeetCode 25)
# ═══════════════════════════════════════
def reverseKGroup(head, k):
    def has_k_nodes(node, k):
        while node and k > 0:
            node, k = node.next, k - 1
        return k == 0

    if not has_k_nodes(head, k):
        return head  # Less than k nodes — no reversal

    prev, cur = None, head
    for _ in range(k):
        nxt = cur.next
        cur.next = prev
        prev, cur = cur, nxt

    # head is now tail of reversed group
    head.next = reverseKGroup(cur, k)
    return prev  # prev is new head of this group

# ═══════════════════════════════════════
# PATTERN 4: Recursive Reversal
# ═══════════════════════════════════════
def reverse_recursive(head):
    if not head or not head.next:
        return head
    new_head = reverse_recursive(head.next)  # Reverse rest
    head.next.next = head  # Make next point back to current
    head.next = None       # Current becomes new tail
    return new_head
🎯 Reversal Interview Insights
  • The 3-pointer reversal (prev, cur, next_node) is the most tested pattern — memorize it cold
  • Always use a dummy node for reverseBetween to avoid head-edge cases
  • reverseKGroup is the hardest reversal problem — interviewers test recursion thinking here
  • Recursive reversal uses O(n) space (call stack) — mention this as a tradeoff

Cycle Detection — Floyd's Algorithm
The tortoise and hare — detect, find entry, and remove cycles
ConceptFloyd's Cycle Detection

Floyd's Algorithm (Tortoise and Hare): Use two pointers — slow moves 1 step, fast moves 2 steps. If there's a cycle, they MUST meet inside it. If fast reaches None, there's no cycle.

💡
Why do they always meet?
Imagine fast is 10 steps ahead of slow inside the cycle. Each step, fast gains... wait, actually slow gains? Fast moves 2, slow moves 1 — RELATIVE speed is 1. So fast catches slow at rate 1 step/iteration. In a cycle of length C, they'll meet within C steps.
CodeCycle Detection + Find Entry Point
Python
# ═══════════════════════════════════════
# STEP 1: Detect if cycle exists — O(n) time, O(1) space
# ═══════════════════════════════════════
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True   # They met → cycle!
    return False  # fast reached None → no cycle

# ═══════════════════════════════════════
# STEP 2: Find where cycle BEGINS — Floyd's Phase 2
# ═══════════════════════════════════════
def find_cycle_start(head):
    slow = fast = head

    # Phase 1: Detect cycle
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            break
    else:
        return None  # No cycle

    # Phase 2: Move one pointer to head
    # Mathematical proof: distance from head to entry
    # = distance from meeting point to entry (within cycle)
    slow = head
    while slow != fast:
        slow = slow.next
        fast = fast.next
    return slow  # They meet at cycle start!

# ═══════════════════════════════════════
# STEP 3: Find cycle LENGTH
# ═══════════════════════════════════════
def cycle_length(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            # Count steps to go around cycle once
            length = 1
            fast = fast.next
            while fast != slow:
                fast = fast.next
                length += 1
            return length
    return 0

# ═══════════════════════════════════════
# STEP 4: Remove cycle
# ═══════════════════════════════════════
def remove_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            break
    else:
        return  # No cycle

    slow = head
    prev = fast
    while slow != fast:
        prev = fast
        slow, fast = slow.next, fast.next
    prev.next = None  # Break the cycle!

Merge & Sort Linked Lists
Merge sort is the optimal O(n log n) sort for linked lists
CodeMerge Two Sorted Lists — LeetCode 21
Python
# ═══════════════════════════════════════
# Iterative: O(n+m) time, O(1) space
# ═══════════════════════════════════════
def mergeTwoLists(l1, l2):
    dummy = Node(0)   # Dummy head — simplifies logic
    cur = dummy

    while l1 and l2:
        if l1.data <= l2.data:
            cur.next = l1
            l1 = l1.next
        else:
            cur.next = l2
            l2 = l2.next
        cur = cur.next

    # Attach remaining nodes
    cur.next = l1 if l1 else l2

    return dummy.next

# ═══════════════════════════════════════
# Recursive version — elegant but O(n+m) stack space
# ═══════════════════════════════════════
def mergeTwoLists_recursive(l1, l2):
    if not l1: return l2
    if not l2: return l1
    if l1.data <= l2.data:
        l1.next = mergeTwoLists_recursive(l1.next, l2)
        return l1
    else:
        l2.next = mergeTwoLists_recursive(l1, l2.next)
        return l2
CodeSort Linked List — Merge Sort O(n log n)
Python
def sortList(head):
    """
    Merge sort for linked lists.
    Time: O(n log n) | Space: O(log n) for recursion
    Why not quick sort? Merge sort is stable and works
    well with linked lists (no random access needed).
    """
    # Base case: 0 or 1 nodes
    if not head or not head.next:
        return head

    # Step 1: Find middle using slow/fast pointers
    slow, fast = head, head.next
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    # Step 2: Split into two halves
    mid = slow.next
    slow.next = None  # Cut the list

    # Step 3: Recursively sort each half
    left = sortList(head)
    right = sortList(mid)

    # Step 4: Merge sorted halves
    return mergeTwoLists(left, right)

# Example: [4→2→1→3] → [1→2→3→4]
💡
Why Merge Sort for Linked Lists?
Merge sort only needs sequential access — perfect for LL. Quick sort needs random access (O(n) per pivot selection in LL). Heapsort needs random access. Merge sort is the natural choice!

Python — LRU Cache O(1) Operations
class LRUNode:
    def __init__(self, key, val):
        self.key = key; self.val = val
        self.prev = self.next = None

class LRUCache:
    """
    Least Recently Used Cache.
    get/put both O(1) using:
    • HashMap: key → node (for O(1) access)
    • Doubly Linked List: maintains recency order
    
    Most Recently Used ← [left_dummy] ↔ ... ↔ [right_dummy] → Least Recently Used
    """
    def __init__(self, capacity):
        self.cap = capacity
        self.cache = {}  # key → node
        # Dummy nodes: left=MRU end, right=LRU end
        self.left = LRUNode(0, 0)   # MRU sentinel
        self.right = LRUNode(0, 0)  # LRU sentinel
        self.left.next = self.right
        self.right.prev = self.left

    def _remove(self, node):
        # Remove from doubly linked list: O(1)
        node.prev.next = node.next
        node.next.prev = node.prev

    def _insert_mru(self, node):
        # Insert right after left dummy (MRU position): O(1)
        node.next = self.left.next
        node.prev = self.left
        self.left.next.prev = node
        self.left.next = node

    def get(self, key):
        if key in self.cache:
            node = self.cache[key]
            self._remove(node)       # Remove from current pos
            self._insert_mru(node)  # Move to MRU position
            return node.val
        return -1

    def put(self, key, val):
        if key in self.cache:
            self._remove(self.cache[key])
        node = LRUNode(key, val)
        self.cache[key] = node
        self._insert_mru(node)
        if len(self.cache) > self.cap:
            # Evict LRU (rightmost real node)
            lru = self.right.prev
            self._remove(lru)
            del self.cache[lru.key]

# Test
cache = LRUCache(2)
cache.put(1, 1)
cache.put(2, 2)
print(cache.get(1))   # 1 (makes 1 MRU)
cache.put(3, 3)       # Evicts 2 (LRU)
print(cache.get(2))   # -1 (was evicted)
print(cache.get(3))   # 3
📚 Deep Dive: Skip List — O(log n) Search in a Linked List ▼

A Skip List is a linked list with multiple levels of "express lanes." Higher levels skip more nodes, allowing O(log n) average search time.

Python — Skip List Concept
# Level 3 (express): head ────────────────────────> 50 → None
# Level 2 (fast):    head ────> 20 ──────────────> 50 → None
# Level 1 (normal):  head → 10 → 20 → 30 → 40 → 50 → None
#
# Search for 40:
# Start at highest level, skip large chunks
# Drop down when overshot → O(log n) average

import random

class SkipNode:
    def __init__(self, val, level):
        self.val = val
        self.next = [None] * (level + 1)  # Array of next ptrs

class SkipList:
    MAX_LEVEL = 4

    def __init__(self):
        self.head = SkipNode(-float('inf'), self.MAX_LEVEL)
        self.level = 0

    def _random_level(self):
        level = 0
        while random.random() < 0.5 and level < self.MAX_LEVEL:
            level += 1
        return level

1. Reverse Linked List
Easy LeetCode 206
Explanation
Solution
Dry Run

Pattern: In-place 3-pointer reversal. Use prev, cur, next_node.

Key: Save next_node = cur.next before overwriting cur.next = prev or you lose the rest of the list!

def reverseList(head):
    prev, cur = None, head
    while cur:
        nxt = cur.next    # Save!
        cur.next = prev   # Reverse
        prev = cur        # Advance prev
        cur = nxt         # Advance cur
    return prev  # New head | O(n) time O(1) space

2. Merge Two Sorted Lists
Easy LeetCode 21
Explanation
Solution

Pattern: Dummy node + two-pointer merge. Compare heads, attach smaller, advance that pointer.

Key trick: Dummy node so you never need to handle "what if the result list is empty" — just return dummy.next.

Python
def mergeTwoLists(l1, l2):
    dummy = cur = Node(0)
    while l1 and l2:
        if l1.val <= l2.val:
            cur.next = l1; l1 = l1.next
        else:
            cur.next = l2; l2 = l2.next
        cur = cur.next
    cur.next = l1 or l2
    return dummy.next  # O(n+m) time, O(1) space

4. Remove Nth Node From End
Medium LeetCode 19
Explanation
Solution

Pattern: Two pointers with n-step gap. Move fast n+1 steps ahead, then move both — when fast reaches None, slow is at the node BEFORE the one to delete.

Python
def removeNthFromEnd(head, n):
    dummy = Node(0)
    dummy.next = head
    fast = slow = dummy
    # Move fast n+1 steps ahead
    for _ in range(n + 1):
        fast = fast.next
    while fast:  # Move both until fast = None
        fast, slow = fast.next, slow.next
    slow.next = slow.next.next  # Delete
    return dummy.next  # O(n) time, O(1) space
5. Palindrome Linked List
Easy LeetCode 234
Explanation
Solution

Pattern: Find middle (slow/fast) → Reverse second half → Compare both halves from outside in.

Brute force: O(n) space with array. Optimal: O(1) space with in-place reversal.

Python
def isPalindrome(head):
    # Step 1: Find middle
    slow = fast = head
    while fast and fast.next:
        slow, fast = slow.next, fast.next.next
    # Step 2: Reverse second half
    prev, cur = None, slow
    while cur:
        cur.next, prev, cur = prev, cur, cur.next
    # Step 3: Compare
    left, right = head, prev
    while right:
        if left.val != right.val: return False
        left, right = left.next, right.next
    return True  # O(n) time, O(1) space
6. Reorder List
Medium LeetCode 143
Explanation
Solution

Problem: L0→L1→...→Ln to L0→Ln→L1→Ln-1→L2→Ln-2→...

Strategy: 3 steps — (1) Find middle (2) Reverse second half (3) Merge alternating.

Python
def reorderList(head):
    if not head or not head.next: return
    # Step 1: Find middle
    slow = fast = head
    while fast and fast.next:
        slow, fast = slow.next, fast.next.next
    # Step 2: Reverse second half
    second = slow.next
    slow.next = None  # Cut
    prev, cur = None, second
    while cur:
        cur.next, prev, cur = prev, cur, cur.next
    second = prev
    # Step 3: Merge alternating
    first = head
    while second:
        tmp1, tmp2 = first.next, second.next
        first.next = second
        second.next = tmp1
        first, second = tmp1, tmp2
7. Merge K Sorted Lists
Hard LeetCode 23
Explanation
Solution

Brute Force: Collect all values, sort, rebuild → O(N log N) time, O(N) space.

Optimal (Min Heap): Push heads of all lists into min heap. Pop min, push its next. O(N log k) where k = number of lists.

Python
import heapq

def mergeKLists(lists):
    dummy = cur = Node(0)
    heap = []
    # Push all list heads with (val, id, node)
    # id breaks ties when vals are equal
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))
    while heap:
        val, i, node = heapq.heappop(heap)
        cur.next = node
        cur = cur.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next
# Time: O(N log k) | Space: O(k)
# N = total nodes, k = number of lists

10. Add Two Numbers
Medium LeetCode 2
Explanation
Solution

Problem: Two non-empty linked lists representing numbers in reverse order. Add and return sum as linked list.

Key: Simulate grade-school addition digit by digit. Track carry!

Python
def addTwoNumbers(l1, l2):
    dummy = cur = Node(0)
    carry = 0
    while l1 or l2 or carry:
        v1 = l1.val if l1 else 0
        v2 = l2.val if l2 else 0
        total = v1 + v2 + carry
        carry, digit = divmod(total, 10)
        cur.next = Node(digit)
        cur = cur.next
        if l1: l1 = l1.next
        if l2: l2 = l2.next
    return dummy.next  # O(max(m,n)) time+space
11. Copy List with Random Pointer
Medium LeetCode 138
Explanation
Solution

Problem: Each node has a random pointer that points to any node or None. Deep copy the entire list.

Trick: Use a hashmap old_node → new_node. Then set next and random pointers.

Python
def copyRandomList(head):
    if not head: return None
    old_to_new = {}  # old node → new node
    # Pass 1: Create all new nodes
    cur = head
    while cur:
        old_to_new[cur] = Node(cur.val)
        cur = cur.next
    # Pass 2: Set next and random pointers
    cur = head
    while cur:
        if cur.next:
            old_to_new[cur].next = old_to_new[cur.next]
        if cur.random:
            old_to_new[cur].random = old_to_new[cur.random]
        cur = cur.next
    return old_to_new[head]  # O(n) time+space
12. Odd Even Linked List
Medium LeetCode 328
Explanation
Solution

Problem: Group all odd-indexed nodes together, then even-indexed. (1-indexed)

Pattern: Split into two lists, reconnect. Use two pointers (odd and even) and maintain even_head.

Python
def oddEvenList(head):
    if not head: return head
    odd = head
    even = even_head = head.next
    while even and even.next:
        odd.next = even.next    # odd jumps 2
        odd = odd.next
        even.next = odd.next   # even jumps 2
        even = even.next
    odd.next = even_head  # Connect odd tail to even head
    return head  # O(n) time, O(1) space
