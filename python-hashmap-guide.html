Chapter 01

Introduction to HashMap

Beginner

A HashMap (called dict in Python) is a data structure that stores key-value pairs. It uses a hash function to compute an index into an array of buckets, enabling near-instant lookups, insertions, and deletions.

Python's dict is one of the most optimized data structures in the language — it powers object attributes, keyword arguments, module namespaces, and more.

How Hashing Works

"name" ──→ hash() ──→ 5765169...1042 ──→ idx % 8 = 2 ──→ bucket[2] ──→ "Alice" "age" ──→ hash() ──→ 3713082...8834 ──→ idx % 8 = 5 ──→ bucket[5] ──→ 30 "city" ──→ hash() ──→ 9182736...4517 ──→ idx % 8 = 7 ──→ bucket[7] ──→ "Hyderabad"

Key Properties

🔑 Unique Keys

Keys must be unique. Duplicate assignment overwrites the previous value.

❄️ Hashable Keys

Keys must be hashable: strings, numbers, tuples (of hashables). Lists and dicts cannot be keys.

📋 Insertion Order

Since Python 3.7+, dicts preserve insertion order as a language guarantee.

🔄 Mutable Values

Values can be any Python object: lists, functions, other dicts, None — anything.

All Ways to Create a Dict

person = {
    "name": "Alice",

pairs = [("a", 1), ("b", 2), ("c", 3)]

# ── 4. From two parallel lists with zip() ────────────────────

d = dict(zip(keys, values))   # {'x': 10, 'y': 20, 'z': 30}

# ── 5. dict.fromkeys() — same default value for all keys ─────
d = dict.fromkeys(["a", "b", "c"], 0)  # {'a': 0, 'b': 0, 'c': 0}
visited = dict.fromkeys(["home", "work"], False)

# ── 6. Empty dict ─────────────────────────────────────────────
empty = {}
empty = dict()

# ── 7. Dict comprehension (covered in detail later) ──────────
squares = {n: n**2 for n in range(1, 6)}  # {1:1, 2:4, 3:9, 4:16, 5:25}

Valid Key Types

Python 
# ✅ Valid (hashable) keys
d = {
    "string_key":        1,
    42:                   2,   # int
    3.14:                 3,   # float
    True:                 4,   # bool (True == 1, careful!)
    (1, 2):               5,   # tuple of hashables
    frozenset({1, 2}):  6,   # frozenset
}

# ❌ Invalid (unhashable) keys — raise TypeError
d[[1, 2]]  = 7   # TypeError: list is unhashable
d[{1:2}]  = 8   # TypeError: dict is unhashable
d[{1,2}]  = 9   # TypeError: set is unhashable

# ⚠️  bool/int overlap gotcha
d = {True: "yes", 1: "one"}
print(d)  # {True: 'one'}  ← 1 overwrites True because True == 1

Core Operations

Beginner

Read, Write, Delete

Python 
user = {"name": "Bob", "age": 25, "role": "dev"}

# ── READ ─────────────────────────────────────────────────────
print(user["name"])                 # "Bob" — raises KeyError if missing
print(user.get("name"))             # "Bob" — returns None if missing
print(user.get("salary", 0))       # 0 — default fallback value

# ── WRITE ────────────────────────────────────────────────────
user["age"] = 26                     # update existing key
user["email"] = "bob@example.com"  # add new key

# ── DELETE ───────────────────────────────────────────────────
del user["role"]                    # removes key; KeyError if missing
val  = user.pop("age")              # removes and returns value
val  = user.pop("salary", None)   # safe pop with default
item = user.popitem()               # removes & returns last (key, val) pair
user.clear()                        # removes all items

# ── MEMBERSHIP ───────────────────────────────────────────────
"name" in user      # True  — key membership (O(1))
"Bob"  in user      # False — checks keys, NOT values!
"Bob"  in user.values()  # True  — value search O(n)

setdefault — Atomic Get-or-Set

Python 
# setdefault: return value if key exists, else set + return default
d = {}
d.setdefault("count", 0)          # sets d["count"] = 0, returns 0
d.setdefault("count", 99)         # key exists → returns 0, d unchanged

# Classic use: grouping into lists
data = [("fruits", "apple"), ("vegs", "carrot"), ("fruits", "mango")]
groups = {}
for category, item in data:
    groups.setdefault(category, []).append(item)
# {'fruits': ['apple', 'mango'], 'vegs': ['carrot']}
💡
Industry tip: Prefer dict.get(key, default) over d[key] when the key's existence is uncertain — avoids try/except overhead and makes intent clear.

Chapter 04

Iteration

Beginner
Python 
inventory = {"apple": 50, "banana": 30, "cherry": 80}

# ── Iterate keys (default) ───────────────────────────────────
for key in inventory:
    print(key)             # apple, banana, cherry

for key in inventory.keys():  # explicit — same result
    print(key)

# ── Iterate values ───────────────────────────────────────────
for qty in inventory.values():
    print(qty)              # 50, 30, 80

# ── Iterate key-value pairs ──────────────────────────────────
for fruit, qty in inventory.items():
    print(f"{fruit}: {qty} units")

# ── Enumerate with index ─────────────────────────────────────
for i, (k, v) in enumerate(inventory.items()):
    print(f"{i}: {k} → {v}")

# ── Sorted iteration ─────────────────────────────────────────
for k in sorted(inventory):
    print(k)

for k, v in sorted(inventory.items(), key=lambda x: x[1], reverse=True):
    print(f"{k}: {v}")        # sorted by qty descending

# ── ⚠️ Never modify dict size while iterating ────────────────
# RuntimeError: dictionary changed size during iteration
# Safe pattern: iterate over a copy
for k in list(inventory.keys()):
    if inventory[k] < 40:
        del inventory[k]

Dict views (.keys(), .values(), .items()) are dynamic — they reflect changes to the dict in real-time. They support set-like operations: d1.keys() & d2.keys() gives the common keys.

View Set Operations

Python

Dict Comprehension

Intermediate
Python 
# ── Syntax ───────────────────────────────────────────────────
# {key_expr: val_expr for item in iterable [if condition]}

# ── Basic ────────────────────────────────────────────────────
squares = {n: n**2 for n in range(1, 6)}

# ── With filter ──────────────────────────────────────────────
even_sq = {n: n**2 for n in range(10) if n % 2 == 0}

# ── Invert a dict (swap key ↔ value) ─────────────────────────
original = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in original.items()}  # {1:'a', 2:'b', 3:'c'}

# ── Transform keys ───────────────────────────────────────────
raw = {"First Name": "Alice", "Last Name": "Smith"}
clean = {k.lower().replace(" ", "_"): v for k, v in raw.items()}
# {'first_name': 'Alice', 'last_name': 'Smith'}

# ── From enumerated list ─────────────────────────────────────
items = ["apple", "banana", "cherry"]
index_map = {v: i for i, v in enumerate(items)}
# {'apple': 0, 'banana': 1, 'cherry': 2}

# ── Conditional value ────────────────────────────────────────
scores = {"Alice": 85, "Bob": 42, "Eve": 91}
grades = {k: "pass" if v >= 50 else "fail" for k, v in scores.items()}
# {'Alice': 'pass', 'Bob': 'fail', 'Eve': 'pass'}

# ── Nested comprehension ─────────────────────────────────────
matrix = {(r, c): r * c for r in range(3) for c in range(3)}
# {(0,0):0, (0,1):0, (1,1):1, (2,2):4, ...}

Chapter 07

Merging & Updating

Intermediate
Python 
defaults = {"theme": "dark",  "lang": "en", "tz": "UTC"}
overrides = {"lang": "te", "tz": "IST"}

# ── Method 1: update() — mutates in-place ────────────────────
config = defaults.copy()
config.update(overrides)

# ── Method 2: ** unpacking — new dict ────────────────────────
config = {**defaults, **overrides}    # Python 3.5+
# right-side keys win on collision

# ── Method 3: | operator — new dict (Python 3.9+) ────────────
config = defaults | overrides         # returns new dict
defaults |= overrides                  # in-place update (3.9+)

# ── Merge multiple dicts ─────────────────────────────────────
result = {**d1, **d2, **d3}
result = d1 | d2 | d3               # Python 3.9+

# ── Deep merge (custom) ──────────────────────────────────────
def deep_merge(base: dict, override: dict) -> dict:
    result = base.copy()
    for key, val in override.items():
        if key in result and isinstance(result[key], dict) and isinstance(val, dict):
            result[key] = deep_merge(result[key], val)
        else:
            result[key] = val
    return result

Nested Dictionaries

Intermediate
Python 
# ── Structure ────────────────────────────────────────────────
org = {
    "engineering": {
        "alice": {"role": "SDE-2", "yoe": 4},
        "bob":   {"role": "SDE-1", "yoe": 1},
    },
    "design": {
        "carol": {"role": "Lead",  "yoe": 6},
    },
}

# ── Access ───────────────────────────────────────────────────
alice_role = org["engineering"]["alice"]["role"]  # "SDE-2"

# ── Safe access with .get() chaining ────────────────────────
role = org.get("hr", {}).get("dave", {}).get("role", "unknown")

# ── Generic deep getter ──────────────────────────────────────
from functools import reduce

def deep_get(d, keys, default=None):
    return reduce(lambda obj, k: obj.get(k, {}) if isinstance(obj, dict) else default, keys, d)

role = deep_get(org, ["engineering", "alice", "role"])  # "SDE-2"

# ── Copy ─────────────────────────────────────────────────────
import copy
shallow = org.copy()           # nested dicts still shared!
deep    = copy.deepcopy(org)  # fully independent clone

collections.defaultdict

Python 
from collections import defaultdict

# Never raises KeyError — creates default on first access
word_count = defaultdict(int)
for word in "the cat sat on the mat".split():
    word_count[word] += 1
# {'the': 2, 'cat': 1, 'sat': 1, 'on': 1, 'mat': 1}

# Group items
groups = defaultdict(list)
for student, grade in [("A","math"), ("B","sci"), ("A","sci")]:
    groups[student].append(grade)
# {'A': ['math', 'sci'], 'B': ['sci']}

# Nested defaultdict
nested = defaultdict(lambda: defaultdict(int))
nested["row1"]["col1"] += 1  # no KeyError, no pre-init needed

collections.Counter

Python 
from collections import Counter

c = Counter("mississippi")
# Counter({'s': 4, 'i': 4, 'p': 2, 'm': 1})

c.most_common(3)       # [('s', 4), ('i', 4), ('p', 2)]

# Arithmetic on Counters
c1 = Counter(a=3, b=1)

print(c1 + c2)           # Counter({'a': 4, 'b': 3})
print(c1 - c2)           # Counter({'a': 2})  — negatives dropped
print(c1 & c2)           # Counter({'a': 1, 'b': 1}) — min
print(c1 | c2)           # Counter({'a': 3, 'b': 2}) — max

# Word frequency — classic interview pattern
words = "to be or not to be".split()
print(Counter(words).most_common(2))  # [('to', 2), ('be', 2)]

Internals & Custom Hashing

Advanced

The hash() Function

Python 
# hash() returns an integer used to index into internal array
hash("hello")      # e.g. 8022161859718... (changes per process)
hash(42)           # 42  (integers hash to themselves)
hash(3.14)         # consistent with numeric equality
hash(True)         # 1  (True == 1, so hash(True) == hash(1))
hash((1, 2, 3))   # tuples hash if all elements are hashable

# HASH RANDOMIZATION: Python randomizes hash seeds per process
# to prevent hash collision DoS attacks. Set PYTHONHASHSEED=0
# for reproducibility in testing/debugging only.

# Equality and hashing contract:
# a == b  →  hash(a) == hash(b)   (must hold!)
# hash(a) == hash(b)  does NOT imply a == b  (collision)

Custom __hash__ and __eq__

Python 
from dataclasses import dataclass

# ── Manual implementation ─────────────────────────────────────
class Point:
    def __init__(self, x: float, y: float):
        self.x, self.y = x, y

    def __eq__(self, other) -> bool:
        if not isinstance(other, Point): return NotImplemented
        return self.x == other.x and self.y == other.y

    def __hash__(self) -> int:
        return hash((self.x, self.y))   # delegate to tuple hash

d = {Point(1, 2): "origin"}
print(d[Point(1, 2)])   # "origin" ✅

# ── Using dataclass (frozen=True makes it hashable) ───────────
@dataclass(frozen=True)
class Coordinate:
    lat:  float
    lon:  float

cache = {Coordinate(17.38, 78.47): "Hyderabad"}  # works!

Python Dict Internals (CPython)

🔢 Compact Arrays

CPython 3.6+ dicts use a compact array + separate hash table indices. Reduces memory by ~20-25%.

📐 Load Factor

Dict resizes when 2/3 full (load factor ≈ 0.67). New size is roughly 4× current entries.

🔗 Open Addressing

Collisions resolved by probing (not chaining). Uses perturbation for pseudo-random probe sequence.

⚡ Key Sharing

Objects of the same class share a single key array (split-table dict), reducing instance memory.

LRU Cache with OrderedDict

Python 
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache    = OrderedDict()

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        self.cache.move_to_end(key)  # mark as recently used
        return self.cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)  # evict LRU

Dispatch Table (replace if-elif chains)

Python 
# ❌ Verbose
def handle(op):
    if   op == "add":    return lambda a,b: a+b
    elif op == "sub":    return lambda a,b: a-b
    elif op == "mul":    return lambda a,b: a*b

# ✅ Dispatch table — O(1) lookup, easy to extend
OPERATIONS: dict[str, callable] = {
    "add": lambda a, b: a + b,
    "sub": lambda a, b: a - b,
    "mul": lambda a, b: a * b,
    "div": lambda a, b: a / b,
}

result = OPERATIONS.get(op, lambda *_: None)(a, b)

Memoization / Caching

Python 
# ── Manual memoization ───────────────────────────────────────
_memo: dict[int, int] = {}

def fib(n: int) -> int:
    if n < 2: return n
    if n not in _memo:
        _memo[n] = fib(n-1) + fib(n-2)
    return _memo[n]

# ── functools.lru_cache (preferred in production) ────────────
from functools import lru_cache, cache

@lru_cache(maxsize=128)
def expensive(n: int) -> int:
    return n ** 3

@cache  # Python 3.9+ — unbounded LRU
def fib(n: int) -> int:
    return n if n < 2 else fib(n-1) + fib(n-2)

Memory & Speed Tips

Python 
# ── 1. Use key membership, not value search ──────────────────
"key" in d             # O(1) ✅
"val" in d.values()   # O(n) ❌ — use a set instead

# ── 2. Pre-size if you know the size ─────────────────────────
# CPython doesn't expose dict pre-allocation in pure Python,
# but building from a list comprehension avoids incremental resizes
d = {k: v for k, v in large_pairs}  # faster than d[k]=v in loop

# ── 3. Avoid repeated key lookups ───────────────────────────
# ❌ Two lookups
if "score" in d:
    x = d["score"]

# ✅ One lookup
x = d.get("score")
if x is not None:  # or  if x is not None
    process(x)

# ── 4. dict vs Counter vs defaultdict ───────────────────────
# Counting elements:
# Counter  > defaultdict(int)  > plain dict  (speed)

# ── 5. Memory: sys.getsizeof ─────────────────────────────────
import sys
print(sys.getsizeof({}))         # ~232 bytes (empty dict)
print(sys.getsizeof({"a":1}))     # ~232 bytes (same — over-allocated)
# Note: getsizeof doesn't count referenced objects!

# ── 6. Thread safety ─────────────────────────────────────────
# Single dict operations (d[k]=v, del d[k]) are thread-safe in CPython
# due to the GIL. But compound operations are NOT:
import threading
lock = threading.Lock()
with lock:
    d[k] = d.get(k, 0) + 1  # safe compound increment

Common Interview Patterns

Python · Interview Patterns 
# ── Two Sum ──────────────────────────────────────────────────
def two_sum(nums: list[int], target: int) -> list[int]:
    seen: dict[int, int] = {}
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    return []

# ── Anagram check ────────────────────────────────────────────
def is_anagram(s: str, t: str) -> bool:
    from collections import Counter
    return Counter(s) == Counter(t)

# ── First unique character ───────────────────────────────────
def first_unique(s: str) -> int:
    from collections import Counter
    freq = Counter(s)
    return next((i for i, c in enumerate(s) if freq[c] == 1), -1)

# ── Subarray sum equals k ────────────────────────────────────
def subarray_sum(nums: list[int], k: int) -> int:
    prefix_count: dict[int, int] = {0: 1}
    total, count = 0, 0
    for n in nums:
        total += n
        count += prefix_count.get(total - k, 0)
        prefix_count[total] = prefix_count.get(total, 0) + 1
    return count

Key gotchas to remember:
1. True, 1, and 1.0 are the same key (equal + same hash).
2. Never delete dict keys while iterating — copy keys first with list(d.keys()).
3. d.get() never raises; d[key] raises KeyError on missing keys.
4. dict.copy() is shallow — nested objects are shared references.
5. Dict views (.keys(), .values(), .items()) reflect live changes.
