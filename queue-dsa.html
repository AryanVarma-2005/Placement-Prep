Introduction to Queues
What is a queue, why it matters, and where you see it every day
Concept What is a Queue?

A queue is a linear data structure that follows the FIFO principle — First In, First Out. Think of it as a line of people waiting: the first person to join the line is the first person to be served.

Just like in real life, a queue has two ends:

  • Front (Head) — where elements leave the queue (DEQUEUE)
  • Rear (Tail) — where new elements join (ENQUEUE)

Queue Implementation in Python
4 ways to implement a queue — from beginner to production-ready
Method 1 Using Python List

The simplest way — use Python's built-in list. However, dequeue is slow! list.pop(0) is O(n) because all elements shift left.

Python
# Queue using Python List
# Simple but dequeue is O(n) — not ideal for large queues

queue = []  # Initialize empty queue

# ENQUEUE: Add to rear (end of list)
queue.append(10)  # Queue: [10]
queue.append(20)  # Queue: [10, 20]
queue.append(30)  # Queue: [10, 20, 30]

print("Queue:", queue)  # [10, 20, 30]

# DEQUEUE: Remove from front (index 0)
# ❌ This is O(n) — all elements shift left!
front = queue.pop(0)
print("Dequeued:", front)  # 10
print("Queue:", queue)     # [20, 30]

# PEEK: View front without removing
if queue:
    print("Front:", queue[0])  # 20

# IS EMPTY
print("Empty?", len(queue) == 0)  # False

# SIZE
print("Size:", len(queue))  # 2
🚨
Why list is bad for queues
pop(0) is O(n) — Python has to shift every remaining element one position to the left. For a queue of 1 million items, that's 999,999 shifts every dequeue!
Method 2 Using collections.deque ✅ Recommended

collections.deque is a doubly linked list under the hood. Both append() (enqueue) and popleft() (dequeue) are O(1). This is what you should use in interviews!

Python
from collections import deque

# Initialize queue
queue = deque()

# ENQUEUE: O(1) — append to right
queue.append(10)  # deque([10])
queue.append(20)  # deque([10, 20])
queue.append(30)  # deque([10, 20, 30])

print(queue)  # deque([10, 20, 30])

# DEQUEUE: O(1) — popleft removes from left
front = queue.popleft()  # Returns 10
print("Dequeued:", front)   # 10
print(queue)              # deque([20, 30])

# PEEK at front (no removal)
print("Front:", queue[0])  # 20

# PEEK at rear
print("Rear:", queue[-1])  # 30

# IS EMPTY
print("Empty?", len(queue) == 0)  # False

# SIZE
print("Size:", len(queue))  # 2

# Initialize with elements directly
q2 = deque([1, 2, 3, 4, 5])
print(q2)  # deque([1, 2, 3, 4, 5])

Method 3 Using queue.Queue (Thread-Safe)

Python's queue.Queue is designed for multithreading. It's thread-safe, meaning multiple threads can safely enqueue/dequeue simultaneously without data corruption.

Python

# Enqueue: put()

q.put(20)
q.put(30)

print("Size:", q.qsize())      # 3
print("Empty?", q.empty())     # False
print("Full?", q.full())       # False (max is 5)

print("Got:", val)  # 10 (FIFO order)

Method 4 Custom Queue Class (Interview Favorite)

Building your own Queue class from scratch — this tests your understanding deeply. Interviewers love this!

Python
from collections import deque

class Queue:
    """Custom Queue implementation using deque internally."""
    
    def __init__(self):
        # Use deque for O(1) operations on both ends
        self._data = deque()
    
    def enqueue(self, val):
        # Add to the REAR of the queue
        self._data.append(val)
    
    def dequeue(self):
        # Remove from the FRONT of the queue
        if self.is_empty():
            raise IndexError("Dequeue from empty queue")
        return self._data.popleft()
    
    def front(self):
        # Peek at front WITHOUT removing
        if self.is_empty():
            raise IndexError("Peek from empty queue")
        return self._data[0]
    
    def rear(self):
        # Peek at rear WITHOUT removing
        if self.is_empty():
            raise IndexError("Rear from empty queue")
        return self._data[-1]
    
    def is_empty(self):
        return len(self._data) == 0
    
    def size(self):
        return len(self._data)
    
    def __repr__(self):
        return f"Queue({list(self._data)})"


# === Testing Our Custom Queue ===
q = Queue()

q.enqueue(5)
q.enqueue(10)
q.enqueue(15)
print(q)                  # Queue([5, 10, 15])
print(q.front())          # 5
print(q.dequeue())        # 5
print(q)                  # Queue([10, 15])
print(q.size())           # 2
print(q.is_empty())       # False

  • Thread-safe? Use queue.Queue — mention this shows system design awareness

Circular Queue
Fix the wasted space problem of linear queues
Problem Why Normal Queues Waste Space

Imagine a fixed-size array queue of size 5. After many enqueue/dequeue operations:

Start:[ _ , _ , _ , _ , _ ] front=0, rear=-1
After 5 enqueues:[ A , B , C , D , E ] front=0, rear=4
Dequeue A,B,C:[ _ , _ , _ , D , E ] front=3, rear=4
Try enqueue F:rear=5 → OUT OF BOUNDS! ❌ "FULL" but 3 spaces wasted!

The left slots are forever lost in a linear queue. Circular Queue fixes this by wrapping rear back to index 0!

🔄 Circular Queue Visualizer
Circular Queue (size=6). Try enqueue/dequeue!
Implementation Circular Queue in Python
Python
class CircularQueue:
    """Fixed-size Circular Queue using array."""
    
    def __init__(self, capacity):
        self.capacity = capacity
        self.queue = [None] * capacity  # Fixed array
        self.front = 0       # Points to front element
        self.rear = -1      # Points to last element
        self._size = 0      # Current number of elements
    
    def is_empty(self):
        return self._size == 0
    
    def is_full(self):
        return self._size == self.capacity
    
    def enqueue(self, val):
        if self.is_full():
            raise OverflowError("Queue is full!")
        # KEY: (rear + 1) % capacity wraps around!
        self.rear = (self.rear + 1) % self.capacity
        self.queue[self.rear] = val
        self._size += 1
    
    def dequeue(self):
        if self.is_empty():
            raise IndexError("Queue is empty!")
        val = self.queue[self.front]
        self.queue[self.front] = None  # Clear slot
        # KEY: (front + 1) % capacity wraps around!
        self.front = (self.front + 1) % self.capacity
        self._size -= 1
        return val
    
    def peek(self):
        if self.is_empty():
            raise IndexError("Queue is empty!")
        return self.queue[self.front]
    
    def __repr__(self):
        result = []
        for i in range(self._size):
            idx = (self.front + i) % self.capacity
            result.append(self.queue[idx])
        return f"CircularQueue({result})"

# Demo
cq = CircularQueue(5)
cq.enqueue(10); cq.enqueue(20); cq.enqueue(30)
print(cq)          # CircularQueue([10, 20, 30])
cq.dequeue()        # Removes 10, front moves to index 1
cq.enqueue(40)     # rear wraps: fills index 0 again!
cq.enqueue(50)
print(cq)          # CircularQueue([20, 30, 40, 50])
print(cq.is_full()) # False (capacity 5, have 4)
💡
The Modulo Magic
The key formula is (index + 1) % capacity. When rear reaches the last index (e.g., 4 in a size-5 queue), adding 1 gives 5, and 5 % 5 = 0 — it wraps back to the start! This is how we "connect" the end to the beginning.

Priority Queue
Not first-come-first-served — highest priority goes first
Concept What is a Priority Queue?

A Priority Queue is like a hospital emergency room — patients are NOT served in order of arrival. The most critically ill patient is treated first, regardless of when they arrived.

Elements have a priority, and the element with the highest priority is served first. In Python, heapq implements a min-heap (lowest number = highest priority).

1
Emergency Surgery — Highest priority, served FIRST
2
Broken Arm — Medium priority
3
Mild Fever — Low priority, waits longest
Python heapq Min-Heap Priority Queue
Python
import heapq

# Python's heapq is a MIN-HEAP
# Smallest element is always at the root (index 0)

heap = []

# Push elements — heapq maintains heap property
heapq.heappush(heap, 30)
heapq.heappush(heap, 10)
heapq.heappush(heap, 50)
heapq.heappush(heap, 5)
heapq.heappush(heap, 20)

print(heap)         # [5, 10, 50, 30, 20] — heap internally
print(heap[0])      # 5 — always the minimum

# Pop min element — O(log n)
print(heapq.heappop(heap))  # 5
print(heapq.heappop(heap))  # 10
print(heapq.heappop(heap))  # 20

# For MAX-HEAP: negate the values!
max_heap = []
for val in [30, 10, 50, 5]:
    heapq.heappush(max_heap, -val)  # Negate!

max_val = -heapq.heappop(max_heap)  # Negate back
print("Max:", max_val)  # 50

# Priority Queue with custom objects: (priority, data)
tasks = []
heapq.heappush(tasks, (3, "Write report"))
heapq.heappush(tasks, (1, "Fix critical bug"))
heapq.heappush(tasks, (2, "Code review"))

while tasks:
    priority, task = heapq.heappop(tasks)
    print(f"[P{priority}] {task}")
# [P1] Fix critical bug
# [P2] Code review
# [P3] Write report

# heapify: Convert list to heap in O(n)
nums = [3, 1, 4, 1, 5, 9]
heapq.heapify(nums)
print(nums[0])  # 1 — min is at top

Monotonic Queue
The secret weapon for sliding window maximum/minimum problems
Concept What is a Monotonic Queue?

A Monotonic Queue maintains its elements in either strictly increasing or decreasing order. When a new element is added, smaller (or larger) existing elements are removed.

🎯
The Key Insight
If we have a window and we're sliding it, old elements that will never be the maximum again are USELESS. The monotonic queue efficiently discards them! This turns O(n·k) to O(n).
📊 Sliding Window Maximum — Interactive

Implementation Sliding Window Maximum
Python
from collections import deque

def maxSlidingWindow(nums, k):
    """
    Find max in every window of size k.
    Time: O(n) — each element pushed/popped at most once
    Space: O(k) — deque stores at most k indices
    """
    if not nums or k == 0:
        return []
    
    dq = deque()  # Stores INDICES, not values
    result = []
    
    for i, num in enumerate(nums):
        
        # Step 1: Remove indices outside current window
        # Window is [i-k+1, i], so any index < i-k+1 is out
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        
        # Step 2: Remove smaller elements from right
        # They can NEVER be the max while current num exists
        while dq and nums[dq[-1]] < num:
            dq.pop()
        
        # Step 3: Add current index
        dq.append(i)
        
        # Step 4: Window is full — record max
        # Front of deque always has index of MAXIMUM
        if i >= k - 1:
            result.append(nums[dq[0]])
    
    return result

# Dry Run
nums = [1, 3, -1, -3, 5, 3, 6, 7]
k = 3
print(maxSlidingWindow(nums, k))
# Output: [3, 3, 5, 5, 6, 7]
# Window [1,3,-1]→3, [3,-1,-3]→3, [-1,-3,5]→5...
🎯 Pattern Recognition: When to Use Monotonic Queue
  • Problem says "find max/min in every window of size k" → Monotonic Queue
  • Problem has "sliding window" + needs range max/min → Monotonic Deque
  • Any problem where you need to efficiently get max/min as a range slides → think monotonic
  • LeetCode patterns: 239, 862, 1438, 1696

Pattern BFS Level Order Traversal Template

This template solves hundreds of interview problems. Memorize it!

Python — BFS Template
from collections import deque

def bfs_template(start):
    """Universal BFS Template — works for trees, graphs, grids."""
    queue = deque([start])
    visited = {start}      # Prevent revisiting
    level = 0
    
    while queue:
        # Process entire level at once
        level_size = len(queue)  # Nodes at current level
        
        for _ in range(level_size):
            node = queue.popleft()
            
            # ✅ PROCESS node here
            
            # Add unvisited neighbors
            for neighbor in get_neighbors(node):
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)
        
        level += 1  # Move to next level
    
    return level


# ===== GRID BFS TEMPLATE =====
def grid_bfs(grid, start_r, start_c, target_r, target_c):
    """BFS on a 2D grid to find shortest path."""
    rows, cols = len(grid), len(grid[0])
    queue = deque([(start_r, start_c, 0)])  # (row, col, distance)
    visited = {(start_r, start_c)}
    directions = [(0,1), (0,-1), (1,0), (-1,0)]  # Right, Left, Down, Up
    
    while queue:
        r, c, dist = queue.popleft()
        
        if r == target_r and c == target_c:
            return dist  # Found! Return shortest distance
        
        for dr, dc in directions:
            nr, nc = r + dr, c + dc
            # Check bounds + not wall + not visited
            if (0 <= nr < rows and 0 <= nc < cols and
                grid[nr][nc] != 1 and (nr, nc) not in visited):
                visited.add((nr, nc))
                queue.append((nr, nc, dist + 1))
    
    return -1  # No path found

Code Rotten Oranges — Classic BFS Problem
🍊
Problem: In a grid, 0=empty, 1=fresh orange, 2=rotten orange. Every minute, rotten oranges spread to adjacent fresh ones. Return minimum minutes to rot all oranges, or -1 if impossible.
Python
from collections import deque

def orangesRotting(grid):
    """
    Multi-source BFS — start from ALL rotten oranges at once!
    Time: O(m*n) | Space: O(m*n)
    """
    rows, cols = len(grid), len(grid[0])
    queue = deque()
    fresh = 0
    
    # Step 1: Find all initially rotten + count fresh
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2:
                queue.append((r, c, 0))  # (row, col, time)
            elif grid[r][c] == 1:
                fresh += 1
    
    # If no fresh oranges, already done
    if fresh == 0:
        return 0
    
    # Step 2: Multi-source BFS from all rotten oranges
    dirs = [(0,1), (0,-1), (1,0), (-1,0)]
    max_time = 0
    
    while queue:
        r, c, time = queue.popleft()
        
        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if (0 <= nr < rows and 0 <= nc < cols
                    and grid[nr][nc] == 1):  # Fresh orange!
                grid[nr][nc] = 2             # Mark as rotten
                fresh -= 1
                max_time = time + 1
                queue.append((nr, nc, time + 1))
    
    # Return -1 if some fresh oranges unreachable
    return max_time if fresh == 0 else -1

# Test
grid = [[2,1,1],[1,1,0],[0,1,1]]
print(orangesRotting(grid))  # 4 minutes

Advanced Queue Concepts
Queue using stacks, stack using queues, and system design concepts
Classic Queue Using Two Stacks

A famous interview problem! Implement a queue using only two stacks. The trick: use one stack for enqueue, another for dequeue. Transfer when needed.

Python
class MyQueue:
    """Queue using two stacks — Amortized O(1) per operation."""
    
    def __init__(self):
        self.in_stack = []   # For enqueue (push here)
        self.out_stack = []  # For dequeue (pop from here)
    
    def push(self, x):
        # Always enqueue to in_stack: O(1)
        self.in_stack.append(x)
    
    def _transfer(self):
        # Move all from in_stack to out_stack
        # This reverses order → FIFO is maintained!
        if not self.out_stack:
            while self.in_stack:
                self.out_stack.append(self.in_stack.pop())
    
    def pop(self):
        # Transfer if out_stack empty, then pop
        self._transfer()
        return self.out_stack.pop()
    
    def peek(self):
        self._transfer()
        return self.out_stack[-1]
    
    def empty(self):
        return not self.in_stack and not self.out_stack

# Dry Run
q = MyQueue()
q.push(1); q.push(2); q.push(3)
# in_stack=[1,2,3], out_stack=[]
print(q.pop())   # Transfer: out=[3,2,1], pop=1
print(q.peek())  # out=[3,2], peek=2
q.push(4)
# in_stack=[4], out_stack=[3,2]
print(q.pop())   # No transfer needed: pop 2
print(q.pop())   # pop 3
print(q.pop())   # Transfer in→out: pop 4
💡
Why Amortized O(1)?
Each element is pushed to in_stack once and popped from out_stack once. Even though transfer looks expensive, across N operations, total work is O(N). Average per operation = O(1). This is amortized analysis!

Producer-Consumer Pattern

import asyncio

async def producer(queue):
    for i in range(5):
        await asyncio.sleep(0.5)  # Simulate work
        await queue.put(i)

async def consumer(queue):
    while True:
        item = await queue.get()  # Waits if empty
        print(f"Consumed: {item}")
        queue.task_done()

async def main():
    queue = asyncio.Queue()
    await asyncio.gather(
        producer(queue),
        consumer(queue)
    )

1. Binary Tree Level Order Traversal
Medium LeetCode 102
Explanation
Solution
Dry Run

Problem: Given root of binary tree, return level order traversal as list of lists.

Pattern: Classic BFS — process entire level before moving to next.

Key Insight: Before processing any node, record len(queue) — this is exactly the number of nodes at the current level!

Python
from collections import deque

def levelOrder(root):
    if not root:
        return []
    
    result = []
    queue = deque([root])
    
    while queue:
        level_size = len(queue)  # Nodes at this level
        level = []
        
        for _ in range(level_size):
            node = queue.popleft()
            level.append(node.val)
            
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        
        result.append(level)
    
    return result
# Time: O(n) | Space: O(n)
Tree: 3 → [9, 20] → [15, 7]
L0:queue=[3] → process 3 → add 9,20result=[[3]]
L1:queue=[9,20] → process both → add 15,7result=[[3],[9,20]]
L2:queue=[15,7] → process both → no childrenresult=[[3],[9,20],[15,7]]
2. Sliding Window Maximum
Hard LeetCode 239
Explanation
Solution

Problem: Given array nums and window size k, return max in each sliding window.

Brute Force: O(n·k) — for each window, scan all k elements. Too slow!

Optimal: Monotonic Decreasing Deque — O(n) total, O(k) space.

Rule: Maintain deque with indices such that nums[dq[0]] ≥ nums[dq[1]] ≥ ... ≥ nums[dq[-1]]. Front is always the max!

Python
from collections import deque

def maxSlidingWindow(nums, k):
    dq = deque()  # Monotonic decreasing deque (stores indices)
    result = []
    
    for i in range(len(nums)):
        # Remove out-of-window indices
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        
        # Remove indices of smaller elements (useless)
        while dq and nums[dq[-1]] < nums[i]:
            dq.pop()
        
        dq.append(i)
        
        if i >= k - 1:
            result.append(nums[dq[0]])
    
    return result
# Time: O(n) | Space: O(k)

4. First Non-Repeating Character in Stream
Medium
Explanation
Solution

Problem: Given a stream of characters, for every character in stream, find first non-repeating character so far.

Pattern: Queue + Counter. Maintain a queue of characters. Use Counter to track frequencies. Front of queue is candidate for non-repeating — if its count > 1, discard it.

Python
from collections import deque, Counter

def firstNonRepeating(stream):
    """For each char in stream, return first non-repeating so far."""
    queue = deque()  # Maintains insertion order
    count = Counter()  # Frequency of each char
    result = []
    
    for char in stream:
        count[char] += 1
        queue.append(char)
        
        # Remove front if it became repeating
        while queue and count[queue[0]] > 1:
            queue.popleft()
        
        # Front of queue is first non-repeating
        result.append(queue[0] if queue else '#')
    
    return result

stream = "abacd"
print(firstNonRepeating(stream))
# a → 'a'
# b → 'a'  (a still first non-repeating)
# a → 'b'  (a repeated, b is now first)
# c → 'b'
# d → 'b'
# Output: ['a', 'a', 'b', 'b', 'b']
5. Task Scheduler
Medium LeetCode 621
Explanation
Solution

Problem: CPU tasks, each with cooldown n between same tasks. Return minimum time to finish all tasks.

Pattern: Max Heap + Queue. Process most frequent task first. Use queue to track tasks in cooldown, release them when cooldown expires.

Python
import heapq
from collections import Counter, deque

def leastInterval(tasks, n):
    """
    Max heap + cooldown queue approach.
    Time: O(n log n) | Space: O(n)
    """
    count = Counter(tasks)
    # Max-heap: negate counts (Python has min-heap)
    heap = [-c for c in count.values()]
    heapq.heapify(heap)
    
    # Queue: stores (-remaining_count, available_at_time)
    cooldown_q = deque()
    time = 0
    
    while heap or cooldown_q:
        time += 1
        
        if heap:
            cnt = heapq.heappop(heap) + 1  # Do task (decrement)
            if cnt:  # Still has remaining count
                cooldown_q.append((cnt, time + n))  # Put in cooldown
        
        # Release tasks whose cooldown expired
        if cooldown_q and cooldown_q[0][1] == time:
            heapq.heappush(heap, cooldown_q.popleft()[0])
    
    return time

print(leastInterval(["A","A","A","B","B","B"], 2))  # 8
6. Moving Average from Data Stream
Easy LeetCode 346
Solution
Python
from collections import deque

class MovingAverage:
    """Moving average using fixed-size deque window."""
    
    def __init__(self, size):
        self.size = size
        self.window = deque()
        self.window_sum = 0
    
    def next(self, val):
        if len(self.window) == self.size:
            # Remove oldest element, subtract from sum
            self.window_sum -= self.window.popleft()
        
        self.window.append(val)
        self.window_sum += val
        
        return self.window_sum / len(self.window)

mv = MovingAverage(3)
print(mv.next(1))  # 1.0  → window=[1]
print(mv.next(10)) # 5.5  → window=[1,10]
print(mv.next(3))  # 4.67 → window=[1,10,3]
print(mv.next(5))  # 6.0  → window=[10,3,5] (1 evicted)
7. Number of Recent Calls

from collections import deque

class RecentCounter:
    """Count requests within last 3000ms using queue."""
    
    def __init__(self):
        self.q = deque()  # Stores timestamps
    
    def ping(self, t: int) -> int:
        self.q.append(t)
        
        # Remove timestamps older than [t-3000, t]
        while self.q[0] < t - 3000:
            self.q.popleft()
        
        return len(self.q)  # All remaining are in range

# Test

print(rc.ping(3002))  # 3 (t=1 removed — 3002-3000=2 > 1)

🔁 Circular Queue Key
  • rear = (rear+1) % capacity
  • front = (front+1) % capacity
  • Full: size == capacity
  • Empty: size == 0
