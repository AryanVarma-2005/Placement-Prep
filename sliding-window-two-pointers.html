01
Introduction
What they are, why they're essential, and how they're related
ConceptWhat is Sliding Window?

A Sliding Window is a contiguous subarray (or substring) that slides over a larger array/string. Instead of recomputing from scratch for every position, you slide the window and update incrementally.

Think of it like reading a book through a magnifying glass — you slide the glass across the page rather than reading each word in isolation.

Array: [2, 3, 5, 1, 4, 6]   ← Window of size 3, sliding right
        [2, 3, 5, 1, 4, 6]   ← Next position
        [2, 3, 5, 1, 4, 6]   ← Next position
        [2, 3, 5, 1, 4, 6]   ← Final position
ConceptWhat are Two Pointers?

Two Pointers uses two index variables that traverse an array, usually from different positions or at different speeds. Together they narrow down a search space from O(n²) to O(n).

Two pointers vs Sliding Window: sliding window is actually a subset of two pointers where both pointers move in the same direction. Two pointers also includes opposite-ends problems (like Two Sum sorted).

02
Fixed Window
Window size k is constant — the simplest sliding window pattern
ConceptFixed Window — The Template

When window size k is given and constant, the algorithm is straightforward:

  1. Build the first window of size k
  2. Slide right: add the new element, remove the element that left
  3. Track the answer at each valid window position
Python — Fixed Window Template
def fixed_window_template(arr, k):
    """Fixed size sliding window. O(n) time, O(1) space."""
    n = len(arr)
    if n < k:
        return -1  # Edge case: array smaller than window

    # Step 1: Build first window
    window_sum = sum(arr[:k])
    result = window_sum  # Initialize answer

    # Step 2: Slide the window
    for i in range(k, n):
        window_sum += arr[i]       # Add incoming element
        window_sum -= arr[i - k]   # Remove outgoing element
        result = max(result, window_sum)  # Update answer

    return result

# Example: Max sum subarray of size 3
# Input: [2, 1, 5, 1, 3, 2], k=3
# Windows: [2,1,5]=8, [1,5,1]=7, [5,1,3]=9, [1,3,2]=6
# Answer: 9
print(fixed_window_template([2,1,5,1,3,2], 3))  # 9

Python
from collections import deque

def firstNegativeInWindow(arr, k):
    """
    Find first negative number in every window of size k.
    Use deque to store indices of negative numbers.
    Time: O(n) | Space: O(k)
    """
    result = []
    dq = deque()  # Stores indices of negatives

    for i in range(len(arr)):
        # Add negative indices to deque
        if arr[i] < 0:
            dq.append(i)

        # Remove indices outside current window
        if dq and dq[0] < i - k + 1:
            dq.popleft()

        # Window is fully formed (i >= k-1)
        if i >= k - 1:
            result.append(arr[dq[0]] if dq else 0)

    return result

arr = [-8, 2, 3, -6, 10]
print(firstNegativeInWindow(arr, 2))
# Windows: [-8,2]=−8, [2,3]=0, [3,-6]=−6, [-6,10]=−6
# Output: [-8, 0, -6, -6]
⚡ Quick Check
In a fixed window of size k, when we slide one step right, what operations do we do?
✅ Add incoming (arr[i]) and subtract outgoing (arr[i-k]). This makes each slide O(1) instead of O(k), turning the full algorithm from O(n·k) to O(n)!
03
Variable Window
Window expands and shrinks based on a condition — the harder, more powerful pattern
ConceptThe Variable Window Idea

In variable window problems, you don't know the size in advance. You expand the right pointer to grow the window, and shrink the left pointer when a condition is violated.

💡
The Core Mental Model
Think of a rubber band stretched across an array. Pull right to expand. When it gets too long (condition violated), push left to shrink. When valid, record the size and pull right again.
TemplateVariable Window — The Universal Template
Python — Variable Window Template
def variable_window_template(arr, condition):
    """
    Universal variable window template.
    Time: O(n) — each element added/removed at most once
    Space: O(1) or O(n) for frequency maps
    """
    left = 0
    result = 0  # Could be max_len, min_len, count, etc.
    window_state = 0  # Sum, count, freq map, etc.

    for right in range(len(arr)):
        # 1. EXPAND: Include arr[right] in window
        window_state += arr[right]

        # 2. SHRINK: While window is invalid, move left
        while not condition(window_state):
            window_state -= arr[left]
            left += 1

        # 3. RECORD: Window is valid, update answer
        result = max(result, right - left + 1)

    return result

# ─────────────────────────────────────────
# VARIANT A: Finding LONGEST valid window
#   → shrink while INVALID (while loop above)
# VARIANT B: Finding SMALLEST valid window
#   → shrink while VALID (record min inside while)
# ─────────────────────────────────────────

def longestSubarrayWithSumK(arr, k):
    """Find length of longest subarray with sum ≤ k."""
    left = window_sum = max_len = 0

    for right in range(len(arr)):
        window_sum += arr[right]  # Expand window

        # Shrink while sum exceeds k
        while window_sum > k:
            window_sum -= arr[left]
            left += 1

        # Window is valid: update max length
        max_len = max(max_len, right - left + 1)

    return max_len

CodeLongest Substring Without Repeating Characters — LC 3
Python
def lengthOfLongestSubstring(s):
    """
    Longest substring with all unique chars.
    Time: O(n) | Space: O(min(n, alphabet_size))
    """
    char_set = set()  # Tracks chars in current window
    left = max_len = 0

    for right in range(len(s)):
        # Shrink window until no duplicate
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1

        char_set.add(s[right])  # Expand
        max_len = max(max_len, right - left + 1)

    return max_len

# Optimized: use dict for O(1) jump
# (skip ahead instead of one-by-one shrink)
def lengthOfLongestSubstring_opt(s):
    last_seen = {}  # char → last index
    left = max_len = 0
    for right, c in enumerate(s):
        if c in last_seen and last_seen[c] >= left:
            left = last_seen[c] + 1  # Jump left past duplicate
        last_seen[c] = right
        max_len = max(max_len, right - left + 1)
    return max_len

print(lengthOfLongestSubstring("abcabcbb"))  # 3 ("abc")
print(lengthOfLongestSubstring("pwwkew"))    # 3 ("wke")

Two Pointers — Core Concept
Two variables, one array — eliminate the inner loop
ConceptThe Two Pointer Idea

Two pointers maintains two index variables to traverse data. There are two main configurations:

↔ Opposite Ends
[← R        L →]
Squeeze toward center.
For sorted arrays.
→→ Same Direction
[L →   R →]
Slow & fast, or window.
For unsorted / streamed.
TemplateTwo Sum in Sorted Array — Classic Template
Python
def twoSumSorted(arr, target):
    """Find pair summing to target in a sorted array.
    Time: O(n) | Space: O(1)
    """
    left, right = 0, len(arr) - 1

    while left < right:
        current_sum = arr[left] + arr[right]

        if current_sum == target:
            return [left + 1, right + 1]  # 1-indexed
        elif current_sum < target:
            left += 1   # Need bigger → move left right
        else:
            right -= 1  # Too big → move right left

    return [-1, -1]  # Not found

# Why it works (sorted array):
# If sum < target: arr[left] too small → left++
# If sum > target: arr[right] too big → right--
# Each iteration eliminates one element → O(n)

Opposite Ends Pattern
Squeeze from both ends — sorted arrays, palindromes, containers
PatternWhen to Use Opposite Ends
  • Array is sorted (or can be sorted)
  • Problem asks for pairs satisfying some condition
  • Problem involves maximizing/minimizing a value using two endpoints
  • Key phrases: "Two Sum", "three sum", "container with water", "trapping rain water"
CodeContainer With Most Water — LC 11
Python
def maxArea(height):
    """
    Container with most water.
    Key insight: always move the SHORTER pointer.
    Moving the taller one can never increase area.
    Time: O(n) | Space: O(1)
    """
    left, right = 0, len(height) - 1
    max_water = 0

    while left < right:
        width = right - left
        water = min(height[left], height[right]) * width
        max_water = max(max_water, water)

        # Move the shorter wall — critical insight!
        if height[left] <= height[right]:
            left += 1
        else:
            right -= 1

    return max_water

print(maxArea([1,8,6,2,5,4,8,3,7]))  # 49
💡
Why move the shorter wall?
Water height is limited by the shorter wall. If you move the taller wall, the height can only stay the same or decrease, but width decreases too → area must decrease. Moving shorter gives chance to find taller wall.
Code3Sum — LC 15
Python
def threeSum(nums):
    """
    Find all unique triplets summing to 0.
    Fix one element, use two pointers for the other two.
    Time: O(n²) | Space: O(1) excluding output
    """
    nums.sort()  # Sort first!
    result = []

    for i in range(len(nums) - 2):
        # Skip duplicates for the fixed element
        if i > 0 and nums[i] == nums[i-1]:
            continue
        # Early termination: smallest triple already > 0
        if nums[i] > 0:
            break

        left, right = i + 1, len(nums) - 1

        while left < right:
            total = nums[i] + nums[left] + nums[right]

            if total == 0:
                result.append([nums[i], nums[left], nums[right]])
                # Skip duplicates for left and right
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                left += 1; right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1

    return result

print(threeSum([-1, 0, 1, 2, -1, -4]))
# [[-1,-1,2],[-1,0,1]]

Python — Same Direction Templates
# ════════════════════════════════════════
# PATTERN 1: Remove Duplicates from Sorted — LC 26
# slow points to last written, fast scans ahead
# ════════════════════════════════════════
def removeDuplicates(nums):
    if not nums: return 0
    slow = 0
    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:  # Found new unique element
            slow += 1
            nums[slow] = nums[fast]  # Write it
    return slow + 1  # O(n) time, O(1) space

# ════════════════════════════════════════
# PATTERN 2: Move Zeros to End — LC 283
# slow = next write position for non-zeros
# ════════════════════════════════════════
def moveZeroes(nums):
    slow = 0  # Next position to write a non-zero
    for fast in range(len(nums)):
        if nums[fast] != 0:
            nums[slow], nums[fast] = nums[fast], nums[slow]
            slow += 1
    # [0,1,0,3,12] → [1,3,12,0,0]

# ════════════════════════════════════════
# PATTERN 3: Sort Colors (Dutch Flag) — LC 75
# THREE pointers: low, mid, high
# ════════════════════════════════════════
def sortColors(nums):
    low = mid = 0
    high = len(nums) - 1

    while mid <= high:
        if nums[mid] == 0:
            nums[low], nums[mid] = nums[mid], nums[low]
            low += 1; mid += 1
        elif nums[mid] == 1:
            mid += 1  # Already in place
        else:  # nums[mid] == 2
            nums[mid], nums[high] = nums[high], nums[mid]
            high -= 1  # Don't increment mid! Not checked yet
    # [2,0,2,1,1,0] → [0,0,1,1,2,2]
CodeTrapping Rain Water — LC 42 (Hard)
Python
def trap(height):
    """
    Water trapped = min(maxL, maxR) - height[i] for each i.
    Two pointer avoids O(n) extra space for prefix maxes.
    Time: O(n) | Space: O(1)
    """
    left, right = 0, len(height) - 1
    left_max = right_max = water = 0

    while left < right:
        if height[left] <= height[right]:
            # Left is bottleneck: water depends on left_max
            if height[left] >= left_max:
                left_max = height[left]
            else:
                water += left_max - height[left]
            left += 1
        else:
            # Right is bottleneck: water depends on right_max
            if height[right] >= right_max:
                right_max = height[right]
            else:
                water += right_max - height[right]
            right -= 1

    return water

print(trap([0,1,0,2,1,0,1,3,2,1,2,1]))  # 6
Key insight for Two-Pointer Trap:
Rule:If height[left] ≤ height[right] → process left side (left_max is definitive)
Rule:Water at a cell = max(0, min(leftMax, rightMax) - height[i])
Why:When height[left] ≤ height[right], right boundary is "tall enough", so left_max is the real constraint

Frequency Maps in Sliding Window
Combine HashMap with window — solve character/element counting problems
ConceptWhy Frequency Maps?

Many sliding window problems track character/element counts rather than sums. A HashMap (Counter) stores frequencies inside the window. When the window slides, you update the count and check validity.

CodeMinimum Window Substring — LC 76 (Hard)
Python
from collections import Counter

def minWindow(s, t):
    """
    Minimum window substring containing all chars of t.
    Time: O(|s| + |t|) | Space: O(|s| + |t|)
    """
    if not t or not s: return ""

    need = Counter(t)   # {char: count we still need}
    missing = len(t)   # Total chars still needed
    best = ""
    left = 0

    for right, c in enumerate(s):
        # Expand: add s[right] to window
        need[c] -= 1
        if need[c] >= 0:  # Was needed (not excess)
            missing -= 1

        # Shrink: while window is valid (has all chars)
        while missing == 0:
            window = s[left:right+1]
            if not best or len(window) < len(best):
                best = window

            need[s[left]] += 1
            if need[s[left]] > 0:  # Was needed, now missing
                missing += 1
            left += 1

    return best

print(minWindow("ADOBECODEBANC", "ABC"))  # "BANC"
Key variables:
needHow many more of each char we need in window
missingTotal characters still needed (shrink when 0)
Expand:need[c]−=1, if ≥0 then missing−=1
Shrink:need[s[left]]+=1, if >0 then missing+=1

Python
from collections import defaultdict

def lengthOfLongestSubstringKDistinct(s, k):
    """Time: O(n) | Space: O(k)"""
    freq = defaultdict(int)
    left = max_len = 0

    for right in range(len(s)):
        freq[s[right]] += 1  # Add to window

        # Shrink while more than k distinct chars
        while len(freq) > k:
            freq[s[left]] -= 1
            if freq[s[left]] == 0:
                del freq[s[left]]
            left += 1

        max_len = max(max_len, right - left + 1)

    return max_len

print(lengthOfLongestSubstringKDistinct("eceba", 2))  # 3 ("ece")
print(lengthOfLongestSubstringKDistinct("aa", 1))     # 2
CodeFind All Anagrams in a String — LC 438

from collections import Counter

def findAnagrams(s, p):
    """Fixed window of size len(p), compare freq maps.
    Time: O(n) | Space: O(1) — at most 26 chars"""
    p_count = Counter(p)
    window = Counter(s[:len(p)])
    result = []

    if window == p_count:
        result.append(0)

    for i in range(len(p), len(s)):
        # Add incoming char
        window[s[i]] += 1
        # Remove outgoing char
        outgoing = s[i - len(p)]
        window[outgoing] -= 1
        if window[outgoing] == 0:
            del window[outgoing]

        if window == p_count:
            result.append(i - len(p) + 1)

    return result

print(findAnagrams("cbaebabacd", "abc"))  # [0, 6]

Monotonic Deque in Sliding Window
Get window max/min in O(1) — the key to hard sliding window problems
ConceptThe Monotonic Deque Idea

A Monotonic Deque (double-ended queue) maintains indices in order such that their corresponding values are strictly decreasing (for max) or increasing (for min).

When a new element arrives that is larger than elements at the back, we pop those back elements — they can never be the maximum while the new element exists in the window.

CodeSliding Window Maximum — LC 239 (Hard)
Python
from collections import deque

def maxSlidingWindow(nums, k):
    """
    Max in every window of size k.
    Time: O(n) — each element pushed/popped at most once
    Space: O(k) — deque stores at most k indices
    """
    dq = deque()  # Stores INDICES, maintains decreasing VALUES
    result = []

    for i in range(len(nums)):
        # 1. Remove indices outside window [i-k+1, i]
        while dq and dq[0] < i - k + 1:
            dq.popleft()

        # 2. Remove indices with smaller values (useless)
        #    They can never be max while nums[i] is in window
        while dq and nums[dq[-1]] < nums[i]:
            dq.pop()

        # 3. Add current index
        dq.append(i)

        # 4. Record max when window is full
        #    Front of deque is always the index of maximum!
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result

# Example: [1,3,-1,-3,5,3,6,7], k=3
# Output:  [3, 3, 5, 5, 6, 7]
print(maxSlidingWindow([1,3,-1,-3,5,3,6,7], 3))
Dry Run: [1,3,-1,-3,5,3,6,7], k=3

i=1:nums[0]=1 < nums[1]=3 → pop 0. dq=[1]
i=2:dq=[1,2] (window full) → result=[nums[1]]=3max=3
i=3:nums[3]=-3, dq=[1,2,3] (i-k+1=1, front ok) → result+=[3]max=3
i=4:nums[4]=5 > all → pop 1,2,3. dq=[4] → result+=[5]max=5
i=5:dq=[4,5] (front 4, i-k+1=3, ok) → result+=[5]max=5
i=6:nums[6]=6 > nums[5]=3 → pop 5. dq=[4,6] → result+=[6]max=6
i=7:nums[7]=7 > nums[6]=6 → pop 6,4. dq=[7] → result+=[7]max=7 ✅

Code4Sum — Generalized k-Sum
Python
def fourSum(nums, target):
    """
    Find all unique quadruplets summing to target.
    Fix two elements, two-pointer for remaining two.
    Time: O(n³) | Space: O(1) excluding output
    """
    nums.sort()
    result = []
    n = len(nums)

    for i in range(n - 3):
        if i > 0 and nums[i] == nums[i-1]: continue  # Skip dup
        for j in range(i+1, n-2):
            if j > i+1 and nums[j] == nums[j-1]: continue  # Skip dup
            left, right = j+1, n-1
            while left < right:
                s = nums[i]+nums[j]+nums[left]+nums[right]
                if s == target:
                    result.append([nums[i],nums[j],nums[left],nums[right]])
                    while leftand nums[left]==nums[left+1]: left+=1
                    while leftand nums[right]==nums[right-1]: right-=1
                    left+=1; right-=1
                elif s < target: left+=1
                else: right-=1
    return result
CodeLongest Repeating Character Replacement — LC 424
Python
def characterReplacement(s, k):
    """
    Longest substring after replacing at most k chars.
    Key: window_size - max_freq <= k means we can fix it.
    Time: O(n) | Space: O(26) = O(1)
    """
    count = {}
    left = max_len = max_freq = 0

    for right in range(len(s)):
        count[s[right]] = count.get(s[right], 0) + 1
        max_freq = max(max_freq, count[s[right]])

        # (window size) - (most frequent char) = chars to replace
        # If > k, shrink window
        window_size = right - left + 1
        if window_size - max_freq > k:
            count[s[left]] -= 1
            left += 1

        max_len = max(max_len, right - left + 1)

    return max_len

print(characterReplacement("AABABBA", 1))  # 4
# Replace one 'B' in "AABA" or "ABBA" to get AAAA/AAAB
💡
The Key Insight
window_size - max_freq = number of characters that need replacing. If this exceeds k, the window is invalid. We never shrink max_freq (only update if larger) — this is intentional for performance.

CodeSmallest Subarray with Sum ≥ Target — LC 209

def minSubArrayLen(target, nums):
    """
    Minimum length subarray with sum >= target.
    Key: record answer INSIDE while loop (when valid).
    Time: O(n) | Space: O(1)
    """
    left = window_sum = 0
    min_len = float('inf')

    for right in range(len(nums)):
        window_sum += nums[right]  # Expand

        # Shrink while window is VALID (sum >= target)
        while window_sum >= target:
            min_len = min(min_len, right - left + 1)  # Record!
            window_sum -= nums[left]
            left += 1

    return min_len if min_len != float('inf') else 0

print(minSubArrayLen(7, [2,3,1,2,4,3]))  # 2 ([4,3])
print(minSubArrayLen(11, [1,1,1,1,1,1,1,1]))  # 0 (impossible)
🎯 The Two Window Patterns Compared
  • Max window: Expand right → shrink left while INVALID → record after while. Example: longest subarray ≤ k.
  • Min window: Expand right → record inside while VALID → shrink left inside while. Example: smallest subarray ≥ k.
  • The difference is where you record — after the while (max) vs inside the while (min).

⚠️
Why variable window is O(n) — not O(n²)
Although there's a while loop inside a for loop, each element is added to the window exactly once (right++) and removed at most once (left++). Total operations = 2n = O(n). The inner while loop doesn't add a multiplier!

1. Maximum Sum Subarray of Size K

Explain

def maxSumK(arr, k):
    win = sum(arr[:k]); best = win
    for i in range(k, len(arr)):
        win += arr[i] - arr[i-k]
        best = max(best, win)

def lengthOfLongestSubstring(s):

3. Container With Most Water
Medium LC 11
Explain
Code

Pattern: Opposite ends two pointers. Always move the shorter wall.

Why: Moving the taller wall can never increase the bounded height, and width decreases → area must decrease.

Python
def maxArea(h):
    l, r = 0, len(h)-1; res = 0
    while l < r:
        res = max(res, min(h[l],h[r])*(r-l))
        if h[l] <= h[r]: l += 1
        else: r -= 1
    return res  # O(n) time, O(1) space

8. Permutation in String
Medium LC 567
Explain
Code

Pattern: Fixed window of len(p1) + frequency map. Check if window freq matches p1 freq.

from collections import Counter
def checkInclusion(p, s):
    need = Counter(p)
    window = Counter(s[:len(p)])
    if window == need: return True
    for i in range(len(p), len(s)):
        window[s[i]] += 1
        out = s[i - len(p)]
        window[out] -= 1
        if not window[out]: del window[out]
        if window == need: return True
    return False  # O(n) time

10. Subarray Product Less Than K
Medium LC 713
Explain
Code

Pattern: Variable window with product. At each right, the number of valid subarrays ending at right = window size (right - left + 1).

Python
def numSubarrayProductLessThanK(nums, k):
    if k <= 1: return 0
    left = count = 0; prod = 1
    for right in range(len(nums)):
        prod *= nums[right]
        while prod >= k:
            prod //= nums[left]; left += 1
        count += right - left + 1  # Subarrays ending at right
    return count  # O(n) time, O(1) space

🎯 Pattern Recognition
  • "Max/min subarray of size k" → Fixed
  • "Longest with condition" → Variable max
  • "Shortest with condition" → Variable min
  • "Two numbers summing to k" (sorted) → Opposite ends
  • "Window max/min" → Monotonic deque
  • "Anagram / permutation" → Fixed + Counter
  • "At most k distinct" → Variable + hashmap
📊 Complexity Summary
  • All sliding window: O(n) time
  • Fixed window space: O(1)
  • Variable (sum): O(1) space
  • Variable (chars): O(k) space
  • Two pointers sorted: O(n), O(1)
  • 3Sum: O(n²) time, O(1) space
  • Monotonic deque: O(n), O(k)
⚠️ Common Mistakes
  • Using nested loops when window applies
  • Forgetting to delete key from Counter when freq=0
  • Wrong shrink condition (invalid vs valid)
  • 3Sum: not skipping duplicates → wrong output
  • Variable window: recording outside shrink = max; inside = min
  • Monotonic deque: forgetting to store INDICES not values
🎯 Interview Phrases That Impress
  • "I'll use a sliding window to reduce from O(n·k) to O(n) — each element is added and removed at most once."
  • "Since the array is sorted, two pointers give O(n) vs O(n²) for the brute force."
  • "I'll track a missing counter to check window validity in O(1) instead of comparing full maps."
  • "A monotonic decreasing deque maintains the maximum of the window — front is always the current max."
  • "For minimum window, I record inside the while loop when the window is valid and shrink it."
