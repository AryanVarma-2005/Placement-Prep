Day 01

String Fundamentals

Understand what a string actually IS before writing a single algorithm.

Concept 1.1

What is a String?

A string is a sequence of characters stored contiguously in memory. Each character is encoded as a number (ASCII or Unicode). In Python, strings are immutable — you cannot change a character in place. Every "modification" creates a brand new string.

⚠️

Immutability is critical! Unlike lists, you CANNOT do s[0] = 'X' on a string. This affects how you solve problems — you often need to convert to a list, modify, then join back.

Concept 1.3

ASCII, ord() & chr() — Numeric Characters

Every character is stored as a number. Knowing ASCII values is essential for many string problems.

💡

Key ASCII values to memorize: 'a' = 97, 'z' = 122, 'A' = 65, 'Z' = 90, '0' = 48, '9' = 57. The difference between lowercase and uppercase of the same letter is always 32.

ascii_operations.py
# ── ord(): character → ASCII number ─────────────────────
print(ord('a'))   # 97
print(ord('z'))   # 122
print(ord('A'))   # 65
print(ord('0'))   # 48

# ── chr(): ASCII number → character ─────────────────────
print(chr(97))    # 'a'
print(chr(65))    # 'A'

# ── Common interview tricks with ASCII ──────────────────

# Index in alphabet (0-based): 'a'→0, 'b'→1, ..., 'z'→25
def char_index(c):
    return ord(c) - ord('a')

print(char_index('a'))   # 0
print(char_index('e'))   # 4
print(char_index('z'))   # 25

# Check if character is lowercase letter
def is_lowercase(c):
    return 'a' <= c <= 'z'    # Pythonic range check

# Convert lowercase to uppercase manually
def to_upper(c):
    return chr(ord(c) - 32)

print(to_upper('a'))   # 'A'

# Shift a character by n positions (Caesar cipher concept)
def shift_char(c, n):
    return chr((ord(c) - ord('a') + n) % 26 + ord('a'))

print(shift_char('a', 3))   # 'd'
print(shift_char('z', 1))   # 'a'  (wraps around!)

# Create frequency array of size 26 (for lowercase letters only)
def freq_array(s):
    freq = [0] * 26
    for c in s:
        freq[ord(c) - ord('a')] += 1
    return freq

print(freq_array("hello"))
# h=1, e=1, l=2, o=1 at their positions

Split & Join — The Power Duo

split() breaks a string into a list. join() merges a list into a string. Together they're one of the most-used string patterns in DSA.

split_join.py
# ── split() ─────────────────────────────────────────────
s = "hello world python"
print(s.split())           # ['hello', 'world', 'python']  (splits on whitespace)
print(s.split(" "))        # ['hello', 'world', 'python']

csv = "a,b,c,d,e"
print(csv.split(","))       # ['a', 'b', 'c', 'd', 'e']
print(csv.split(",", 2))    # ['a', 'b', 'c,d,e']  (max 2 splits)

# splitlines: split on newlines
multiline = "line1\nline2\nline3"
print(multiline.splitlines())  # ['line1', 'line2', 'line3']

# ── join() ──────────────────────────────────────────────
words = ["hello", "world"]
print(" ".join(words))        # "hello world"
print("-".join(words))        # "hello-world"
print("".join(words))         # "helloworld"
print(", ".join(words))       # "hello, world"

# ── Classic Pattern: reverse words in sentence ──────────
def reverse_words(s):
    words = s.strip().split()
    return " ".join(reversed(words))

print(reverse_words("the sky is blue"))
# "blue is sky the"

# ── Efficient string building (avoid += in loops!) ──────
# WRONG (O(n²) due to new string each time):
# result = ""
# for c in some_list:
#     result += c

# CORRECT (O(n) — collect then join):
chars = []
for c in "hello":
    chars.append(c.upper())
result = "".join(chars)   # "HELLO"
⚠️

Never use result += char inside a loop! Each += creates a new string object in memory — that's O(n²) total. Use a list and "".join() at the end instead. This is one of the most common Python performance mistakes.

Concept 2.3

Reverse a String — All Methods

reverse_string.py
s = "hello"

# ── Method 1: Slicing (FASTEST, Pythonic) ───────────────
reversed_s = s[::-1]         # "olleh"

# ── Method 2: reversed() + join ─────────────────────────
reversed_s = "".join(reversed(s))

# ── Method 3: List conversion (useful if you need to modify) ─
chars = list(s)
chars.reverse()
reversed_s = "".join(chars)

# ── Method 4: Two-pointer (manual, good to know) ────────
def reverse_string(s):
    chars = list(s)
    left, right = 0, len(chars) - 1
    while left < right:
        chars[left], chars[right] = chars[right], chars[left]
        left += 1; right -= 1
    return "".join(chars)

# ── Reverse only vowels ─────────────────────────────────
def reverse_vowels(s):
    vowels = set("aeiouAEIOU")
    chars = list(s)
    left, right = 0, len(s) - 1
    while left < right:
        while left < right and chars[left] not in vowels:
            left += 1
        while left < right and chars[right] not in vowels:
            right -= 1
        chars[left], chars[right] = chars[right], chars[left]
        left += 1; right -= 1
    return "".join(chars)

print(reverse_vowels("hello"))   # "holle"
Day 03

Two Pointers on Strings

The two-pointer pattern solves palindromes, comparisons, and subsequence problems elegantly.

Pattern 3.1

Palindrome Check

A string reads the same forwards and backwards. Two pointers from both ends, move inward, compare.

palindrome.py
# ── Basic palindrome check ──────────────────────────────
def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1; right -= 1
    return True

# Pythonic one-liner
def is_palindrome_v2(s):
    return s == s[::-1]

print(is_palindrome("racecar"))    # True
print(is_palindrome("hello"))      # False

# ── Valid Palindrome (ignore spaces, punctuation, case) ─
def valid_palindrome(s):
    # filter: keep only alphanumeric, lowercase
    clean = [c.lower() for c in s if c.isalnum()]
    left, right = 0, len(clean) - 1
    while left < right:
        if clean[left] != clean[right]:
            return False
        left += 1; right -= 1
    return True

print(valid_palindrome("A man, a plan, a canal: Panama"))  # True
print(valid_palindrome("race a car"))                    # False

# ── Valid Palindrome II: delete at most one character ────
def valid_palindrome_ii(s):
    def check(left, right):
        while left < right:
            if s[left] != s[right]:
                return False
            left += 1; right -= 1
        return True

    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            # Try skipping left OR skipping right
            return check(left + 1, right) or check(left, right - 1)
        left += 1; right -= 1
    return True

print(valid_palindrome_ii("abca"))   # True (delete 'c')
print(valid_palindrome_ii("abc"))    # False
Pattern 3.2

Is Subsequence

Check if string s is a subsequence of string t. A subsequence maintains order but doesn't need to be contiguous.

subsequence.py
# ── Is Subsequence (two pointer) ────────────────────────
def is_subsequence(s, t):
    i = j = 0
    while i < len(s) and j < len(t):
        if s[i] == t[j]:
            i += 1   # matched! advance s pointer
        j += 1       # always advance t pointer
    return i == len(s)   # did we match all of s?

print(is_subsequence("ace", "abcde"))   # True
print(is_subsequence("aec", "abcde"))   # False (order matters!)

# ── Compare strings with backspace (#) ──────────────────
def backspace_compare(s, t):
    def process(string):
        result = []
        for c in string:
            if c == '#':
                if result: result.pop()
            else:
                result.append(c)
        return result
    return process(s) == process(t)

print(backspace_compare("ab#c", "ad#c"))   # True  (both → "ac")
print(backspace_compare("a##c", "#a#c"))   # True  (both → "c")

# ── Longest common prefix ───────────────────────────────
def longest_common_prefix(strs):
    if not strs: return ""
    prefix = strs[0]
    for s in strs[1:]:
        while not s.startswith(prefix):
            prefix = prefix[:-1]       # shrink prefix from right
            if not prefix: return ""
    return prefix

print(longest_common_prefix(["flower","flow","flight"]))
# "fl"
Day 04

Sliding Window on Strings

The most important pattern for substring problems. Variable window driven by character frequency constraints.

Pattern 4.1

Longest Substring Without Repeating Characters

Classic variable sliding window. Expand right, shrink left when we get a duplicate. Track with a set or dictionary.

no_repeat.py
# ── Longest Substring Without Repeating Chars ───────────
def length_of_longest_substring(s):
    char_index = {}   # char → last seen index
    max_len = 0
    left = 0

    for right, ch in enumerate(s):
        if ch in char_index and char_index[ch] >= left:
            # duplicate found! shrink left past the duplicate
            left = char_index[ch] + 1
        char_index[ch] = right
        max_len = max(max_len, right - left + 1)

    return max_len

print(length_of_longest_substring("abcabcbb"))  # 3 ("abc")
print(length_of_longest_substring("bbbbb"))      # 1 ("b")
print(length_of_longest_substring("pwwkew"))     # 3 ("wke")

# ── Alternative using set (simpler but slightly slower) ─
def longest_no_repeat_set(s):
    char_set = set()
    left = max_len = 0
    for right in range(len(s)):
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        char_set.add(s[right])
        max_len = max(max_len, right - left + 1)
    return max_len
Pattern 4.2

Longest Substring with K Distinct Characters

k_distinct.py
from collections import defaultdict

def longest_k_distinct(s, k):
    freq = defaultdict(int)
    left = max_len = 0

    for right in range(len(s)):
        freq[s[right]] += 1          # expand window

        while len(freq) > k:           # too many distinct chars
            freq[s[left]] -= 1
            if freq[s[left]] == 0:
                del freq[s[left]]
            left += 1                  # shrink window

        max_len = max(max_len, right - left + 1)

    return max_len

print(longest_k_distinct("araaci", 2))   # 4 ("araa")
print(longest_k_distinct("cbbebi", 3))   # 5 ("cbbeb")
Pattern 4.3

Minimum Window Substring — The Hard One

Find the smallest window in s that contains all characters of t. Use two frequency maps and a formed counter.

min_window.py
from collections import Counter

def min_window(s, t):
    if not t or not s: return ""

    need = Counter(t)        # required frequency of each char
    have = {}                # current window frequency
    formed = 0              # how many chars are "satisfied"
    required = len(need)    # how many distinct chars we need
    left = 0
    result = ""
    min_len = float('inf')

    for right in range(len(s)):
        ch = s[right]
        have[ch] = have.get(ch, 0) + 1

        # Check if this char is now fully satisfied
        if ch in need and have[ch] == need[ch]:
            formed += 1

        # Shrink window from left while all chars satisfied
        while formed == required:
            window_size = right - left + 1
            if window_size < min_len:
                min_len = window_size
                result = s[left:right + 1]

            left_ch = s[left]
            have[left_ch] -= 1
            if left_ch in need and have[left_ch] < need[left_ch]:
                formed -= 1
            left += 1

    return result

print(min_window("ADOBECODEBANC", "ABC"))   # "BANC"
print(min_window("a", "a"))                 # "a"
print(min_window("a", "aa"))                # ""
Pattern 4.4

Find All Anagrams in a String

anagram_positions.py
from collections import Counter

def find_anagrams(s, p):
    if len(p) > len(s): return []

    p_count = Counter(p)
    window = Counter(s[:len(p)])
    result = []
    k = len(p)

    if window == p_count:
        result.append(0)

    for i in range(1, len(s) - k + 1):
        # Add new char
        new_ch = s[i + k - 1]
        window[new_ch] += 1
        # Remove old char
        old_ch = s[i - 1]
        window[old_ch] -= 1
        if window[old_ch] == 0:
            del window[old_ch]

        if window == p_count:
            result.append(i)

    return result

print(find_anagrams("cbaebabacd", "abc"))   # [0, 6]
print(find_anagrams("abab", "ab"))           # [0, 1, 2]

Concept 5.1

Frequency Maps — The Foundation

frequency.py
from collections import Counter, defaultdict

# ── Three ways to build a frequency map ─────────────────
s = "hello world"

# Method 1: Counter (best for most cases)
freq = Counter(s)
print(freq)
# Counter({'l': 3, 'o': 2, 'h': 1, 'e': 1, ' ': 1, 'w': 1, 'r': 1, 'd': 1})
print(freq['l'])          # 3
print(freq['z'])          # 0  (no KeyError!)
print(freq.most_common(3)) # top 3 most frequent

# Method 2: defaultdict
freq2 = defaultdict(int)
for c in s:
    freq2[c] += 1

# Method 3: Manual dict
freq3 = {}
for c in s:
    freq3[c] = freq3.get(c, 0) + 1

# ── First unique character ───────────────────────────────
def first_unique_char(s):
    freq = Counter(s)
    for i, c in enumerate(s):
        if freq[c] == 1:
            return i
    return -1

print(first_unique_char("leetcode"))   # 0 ('l')
print(first_unique_char("loveleetcode"))  # 2 ('v')
print(first_unique_char("aabb"))       # -1 (none unique)
Concept 5.2

Anagram Detection & Grouping

anagrams.py
from collections import Counter, defaultdict

# ── Check if two strings are anagrams ───────────────────
def is_anagram(s, t):
    if len(s) != len(t): return False
    return Counter(s) == Counter(t)

# Alternatively: sorted strings must be equal
def is_anagram_v2(s, t):
    return sorted(s) == sorted(t)   # O(n log n)

print(is_anagram("anagram", "nagaram"))   # True
print(is_anagram("rat", "car"))           # False

# ── Group Anagrams ──────────────────────────────────────
def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))   # canonical form: sorted chars
        groups[key].append(s)
    return list(groups.values())

words = ["eat","tea","tan","ate","nat","bat"]
print(group_anagrams(words))
# [['eat','tea','ate'], ['tan','nat'], ['bat']]

# ── Isomorphic Strings ──────────────────────────────────
# s and t are isomorphic if chars can be mapped 1-to-1
def is_isomorphic(s, t):
    s_to_t = {}
    t_to_s = {}
    for cs, ct in zip(s, t):
        if cs in s_to_t:
            if s_to_t[cs] != ct: return False
        else:
            if ct in t_to_s: return False
            s_to_t[cs] = ct
            t_to_s[ct] = cs
    return True

print(is_isomorphic("egg", "add"))    # True  (e→a, g→d)
print(is_isomorphic("foo", "bar"))    # False (o can't map to both a and r)

# ── Word Pattern ────────────────────────────────────────
def word_pattern(pattern, s):
    words = s.split()
    if len(pattern) != len(words): return False
    p_to_w, w_to_p = {}, {}
    for p, w in zip(pattern, words):
        if p_to_w.get(p, w) != w: return False
        if w_to_p.get(w, p) != p: return False
        p_to_w[p] = w; w_to_p[w] = p
    return True

print(word_pattern("abba", "dog cat cat dog"))   # True
print(word_pattern("abba", "dog cat cat fish"))  # False
Day 06

Dynamic Programming & Advanced

The hardest string problems. DP, KMP pattern matching, and string encoding — the big leagues.

Advanced 6.1

Longest Common Subsequence (LCS)

Given two strings, find the length of the longest subsequence present in both. Classic 2D DP problem.

📝

DP Idea: Build a table where dp[i][j] = LCS of first i chars of s1 and first j chars of s2. If chars match, extend the diagonal. Otherwise, take the max of left or top cell.

lcs.py
# ── Longest Common Subsequence ──────────────────────────
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    # dp[i][j] = LCS length of s1[:i] and s2[:j]
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:      # chars match!
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    return dp[m][n]

print(lcs("abcde", "ace"))     # 3 ("ace")
print(lcs("abc", "abc"))       # 3
print(lcs("abc", "def"))       # 0
Advanced 6.2

Longest Palindromic Substring

Find the longest substring that is a palindrome. Two approaches: Expand Around Center O(n²) or Manacher's algorithm O(n).

longest_palindrome.py
# ── Expand Around Center — O(n²) ────────────────────────
def longest_palindrome(s):
    if not s: return ""

    start = end = 0

    def expand(left, right):
        while left >= 0 and right < len(s) and s[left] == s[right]:
            left -= 1; right += 1
        return left + 1, right - 1   # last valid positions

    for i in range(len(s)):
        # Odd length palindromes (center at i)
        l1, r1 = expand(i, i)
        # Even length palindromes (center between i and i+1)
        l2, r2 = expand(i, i + 1)

        if r1 - l1 > end - start: start, end = l1, r1
        if r2 - l2 > end - start: start, end = l2, r2

    return s[start:end + 1]

print(longest_palindrome("babad"))   # "bab" or "aba"
print(longest_palindrome("cbbd"))    # "bb"
print(longest_palindrome("racecar")) # "racecar"

# ── Count all palindromic substrings ────────────────────
def count_palindromes(s):
    count = 0

    def expand_count(l, r):
        c = 0
        while l >= 0 and r < len(s) and s[l] == s[r]:
            c += 1; l -= 1; r += 1
        return c

    for i in range(len(s)):
        count += expand_count(i, i)      # odd
        count += expand_count(i, i + 1)  # even

    return count

print(count_palindromes("abc"))     # 3  (a, b, c)
print(count_palindromes("aaa"))     # 6  (a,a,a,aa,aa,aaa)
Advanced 6.3

Edit Distance (Levenshtein)

Minimum number of operations (insert, delete, replace) to convert one string to another. Classic DP.

edit_distance.py
def edit_distance(word1, word2):
    m, n = len(word1), len(word2)
    # dp[i][j] = min ops to convert word1[:i] to word2[:j]
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # Base cases: empty string conversions
    for i in range(m + 1): dp[i][0] = i  # delete all chars of word1
    for j in range(n + 1): dp[0][j] = j  # insert all chars of word2

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i-1] == word2[j-1]:    # chars match: free!
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(
                    dp[i-1][j],    # delete from word1
                    dp[i][j-1],    # insert into word1
                    dp[i-1][j-1]  # replace
                )
    return dp[m][n]

print(edit_distance("horse", "ros"))    # 3
print(edit_distance("intention", "execution"))  # 5
print(edit_distance("", "abc"))         # 3
Advanced 6.4

KMP Pattern Matching — O(n+m)

Naive pattern search is O(n×m). KMP builds a failure function to skip redundant comparisons, achieving O(n+m).

kmp.py
# ── KMP: Knuth-Morris-Pratt Pattern Matching ────────────

def build_lps(pattern):
    """Build Longest Proper Prefix-Suffix array"""
    m = len(pattern)
    lps = [0] * m
    length = 0   # length of previous longest prefix-suffix
    i = 1

    while i < m:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                length = lps[length - 1]  # fall back
            else:
                lps[i] = 0
                i += 1
    return lps

def kmp_search(text, pattern):
    """Returns all starting indices where pattern is found in text"""
    n, m = len(text), len(pattern)
    if m == 0: return []

    lps = build_lps(pattern)
    results = []
    i = j = 0

    while i < n:
        if text[i] == pattern[j]:
            i += 1; j += 1
        if j == m:                      # full match found!
            results.append(i - j)
            j = lps[j - 1]             # use LPS to continue
        elif i < n and text[i] != pattern[j]:
            if j != 0:
                j = lps[j - 1]         # skip using LPS
            else:
                i += 1
    return results

text = "AABAACAADAABAABA"
pat  = "AABA"
print(kmp_search(text, pat))   # [0, 9, 12]

text2 = "aababab"
pat2  = "abab"
print(kmp_search(text2, pat2)) # [1, 3]

String Encode & Decode

Design a system to encode a list of strings to a single string and decode back. Use length-prefix encoding.

encode_decode.py
# ── Encode: ["hello","world"] → "5#hello5#world" ────────
def encode(strs):
    result = ""
    for s in strs:
        result += str(len(s)) + "#" + s
    return result

# ── Decode: "5#hello5#world" → ["hello","world"] ────────
def decode(s):
    result = []
    i = 0
    while i < len(s):
        j = s.index("#", i)        # find next '#'
        length = int(s[i:j])        # read length
        word = s[j+1:j+1+length]   # extract word
        result.append(word)
        i = j + 1 + length          # advance past word
    return result

words = ["hello", "world", "#special", ""]
encoded = encode(words)
print(encoded)                  # "5#hello5#world8##special0#"
print(decode(encoded) == words)  # True

Pattern Guide

See the Problem → Know the Technique

🔍 Substring / window problem?

→ Sliding window with a frequency map. Variable window for "at most k" or "smallest". Fixed window otherwise.

🔄 Palindrome check?

→ Two pointers from both ends. Or expand from center for "find longest palindromic substring".

🔤 Anagram / permutation?

→ Sort both strings and compare, or use Counter(). For "in string" → sliding window of fixed size.

🔢 Frequency / count?

→ Counter or defaultdict(int). For "only lowercase letters" use a freq array of size 26.

📏 Two strings comparison?

→ Two pointers, one on each string. Move both or one based on matching condition.

✏️ Minimum edits / conversions?

→ 2D dynamic programming. Fill table bottom-up. Choices: match (diagonal), insert, delete, replace.

🔎 Pattern search in text?

→ Python's built-in .find() or .index(). For performance-critical: KMP algorithm O(n+m).

🗂️ Group/classify strings?

→ Use a canonical key (sorted string, sorted character tuple) as hash map key to group anagrams/similar strings.
